# 支持

Docusaurus 拥有一个由数千名开发者组成的社区。

本页面列出了您可以参与的 Docusaurus 相关社区；请查阅本节其他页面以获取更多在线及线下学习资源。

在参与 Docusaurus 社区之前，[请阅读我们的行为准则](https://engineering.fb.com/codeofconduct/)。我们采用了《贡献者公约》([Contributor Covenant](https://www.contributor-covenant.org/))，期望所有社区成员遵守其中的准则。

## Stack Overflow {#stack-overflow}

Stack Overflow 是提出代码层面问题或解决特定错误的常用论坛。浏览标记为 **docusaurus** 的[现有问题](https://stackoverflow.com/questions/tagged/docusaurus)或[提出您的问题](https://stackoverflow.com/questions/ask?tags=docusaurus)！

## 讨论论坛 {#discussion-forums}

有许多在线论坛可用于讨论最佳实践、应用架构以及 Docusaurus 的未来发展。如果您有可解答的代码问题，通常 Stack Overflow 更为适合。

- [Docusaurus 在线聊天](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) 获取用户帮助
  - [#contributors](https://discord.gg/6g6ASPA) 获取贡献帮助
- [Reddit 上的 Docusaurus 社区](https://www.reddit.com/r/docusaurus/)

## 功能请求 {#feature-requests}

对于新功能请求，您可以在我们的[功能请求看板 (Canny)](/feature-requests)上发帖。该工具便于路线规划，并支持按投票排序，使核心团队能更清晰地了解哪些功能需求较高，相较于 GitHub 问题更易于分类。请避免直接提交新功能（尤其是大型功能）的拉取请求，因为可能已有人正在开发或该功能已列入我们的路线图。请先与我们沟通！

## 新闻 {#news}

获取 Docusaurus 的最新动态，请关注 X 上的 [**@docusaurus**](https://x.com/docusaurus) 以及本网站的[官方 Docusaurus 博客](/blog)。
