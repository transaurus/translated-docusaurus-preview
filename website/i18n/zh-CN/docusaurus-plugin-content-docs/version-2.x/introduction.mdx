# 简介

⚡️ Docusaurus 助您快速搭建**美观的文档站点**。

💸 自建技术栈成本高昂。您只需专注于**内容创作**，撰写 Markdown 文件即可。

💥 需要更多功能？可使用**高级特性**如版本控制、国际化、搜索和主题定制。

💅 查看**[精选 Docusaurus 站点](/showcase?tags=favorite)**获取灵感，阅读**[用户评价](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**静态站点生成器**。它构建**单页应用**，通过快速的客户端导航和完整的**React**交互能力，提供开箱即用的**文档功能**，也能创建**各类网站**（个人网站、产品页、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速上手 ⏱️ {#fast-track}

通过实践**5分钟**了解 Docusaurus！

新建 Docusaurus 站点并完成**简短**的内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 后创建新站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

访问 [`http://localhost:3000`](http://localhost:3000) 跟随教程。

:::tip

立即通过 **[docusaurus.new](https://docusaurus.new)** 在线体验 Docusaurus！

或阅读 **[5分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2 是完全基于现代化工具链的重构版本。[v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0)后，我们强烈建议您**使用 v2 而非已弃用的 v1**。

已有[大量用户](/showcase)采用 Docusaurus v2（[趋势数据](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**适用 Docusaurus v2 的场景：**

- :white_check_mark: 需要现代化的 Jamstack 文档站点
- :white_check_mark: 需要支持客户端路由的单页应用 (SPA)
- :white_check_mark: 需要完整的 React 和 MDX 能力
- :white_check_mark: 无需兼容 IE11

**适用 [Docusaurus v1](https://v1.docusaurus.io/) 的场景：**

- :x: 不需要单页应用 (SPA)
- :x: 需要兼容 IE11（注意：IE [已终止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

对于现有v1用户想要升级到v2版本，您可以遵循我们的[迁移指南](./migration/migration-overview.mdx)。

## 功能特性 {#features}

Docusaurus在开发者和贡献者体验方面投入了高度关注。

- ⚛️ **基于💚和React构建**：
  - 使用React进行扩展和定制
  - 通过提供自己的React组件，完全掌控网站的浏览体验
- **可插拔**：
  - 使用基础模板快速启动网站，然后利用高级功能和插件
  - 将插件开源并与社区分享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口点，便于贡献者维护
  - 热重载，变更时极速增量构建
  - 基于路由的代码和数据分割
  - 轻松部署到GitHub Pages、Netlify、Vercel等服务

我们的共同目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建出色的文档网站。

- 🎯 **SEO友好**：
  - 为每个可能的路径静态生成HTML文件。
  - 页面专属SEO优化，帮助用户直接找到与其问题相关的官方文档。
- 📝 **基于MDX**：
  - 在Markdown中嵌入JSX和React编写交互式组件。
  - 通过实时编辑器分享代码，让用户即刻爱上您的产品。
- 🔍 **搜索**：全站内容可搜索。
- 💾 **文档版本控制**：保持文档与项目版本同步。
- 🌍 **国际化(i18n)**：支持多语言站点翻译。

Docusaurus 2生来就是为了让所有用户都能无障碍使用，且速度极快。

- ⚡️ **极速加载**。Docusaurus 2遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容加载如闪电般迅速。
- 🦖 **无障碍设计**。注重可访问性，让所有用户都能平等访问您的网站。

## 设计原则 {#design-principles}

- **学习成本低**。Docusaurus应该易于学习和使用，因为其API非常精简。即使需要编写更多代码和时间，用户仍能实现大多数功能。没有抽象比错误的抽象更好，我们不希望用户不得不绕过错误的抽象。必看演讲——[最小API表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直观**。用户在查看Docusaurus项目目录或添加新功能时不会感到不知所措。它应该看起来直观且易于构建，使用他们熟悉的方法。
- **分层架构**。我们堆栈的每一层（内容/主题/样式）之间的关注点分离应该清晰——抽象良好且模块化。
- **合理的默认值**。常见和流行的性能优化和配置将为用户完成，但他们可以选择覆盖它们。
- **无供应商锁定**。用户不需要使用默认插件或CSS，尽管我们强烈建议这样做。某些核心基础设施如React Loadable和React Router不可替换，因为我们默认对它们进行性能优化，但更高层次的可以。Markdown引擎、CSS框架、CSS方法和其他架构的选择完全由用户决定。

我们相信，作为开发者，了解库的工作原理有助于我们更好地使用它。因此，我们致力于解释Docusaurus的架构和各种组件，希望用户在阅读后能更深入地理解该工具，并更熟练地使用它。

## 与其他工具的比较 {#comparison-with-other-tools}

在所有静态网站生成器中，Docusaurus独特地专注于文档网站，并提供了许多开箱即用的功能。

我们还研究了其他主流静态站点生成器，并希望在此分享我们的对比分析，帮助您在众多选择中找到最适合的方案。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/)功能丰富，拥有庞大的插件生态系统，能够实现Docusaurus的所有功能。但相应地，其学习曲线更为陡峭。Gatsby作为通用型框架适合构建各类网站，而Docusaurus则专注于将文档编写与发布做到极致。

虽然GraphQL并非构建Gatsby站点的必要条件，但它仍是Gatsby的核心特性。对于大多数静态网站场景，您可能并不需要GraphQL提供的灵活性。

Docusaurus 2的许多设计灵感源自Gatsby的精华特性，它同样是个优秀的替代选择。

[Docz](https://github.com/pedronauck/docz)是基于Gatsby的主题系统构建的文档工具，目前功能完善度不及Docusaurus。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/)是另一款流行的混合式React框架。虽然可用于构建文档网站，但其设计并非专为文档场景优化，实现Docusaurus开箱即用的功能需要更多开发工作。

[Nextra](https://github.com/shuding/nextra)是基于Next.js的静态站点生成器，目前功能完备性低于Docusaurus。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/)与Docusaurus有许多相似之处——都专注于内容型网站并提供开箱即用的文档功能。区别在于VuePress基于Vue技术栈，而Docusaurus基于React。如需Vue解决方案，VuePress是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/)是流行的Python静态站点生成器，其核心价值与Docusaurus相似。

如果您不需要单页应用且不计划使用React技术栈，这是个理想选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/)提供了精美的主题方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/)能快速创建文档网站，但并非静态站点生成器且对SEO不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/)拥有简洁的设计风格，曾被众多开源项目采用。随着其产品逐渐转向商业化，许多功能已不再符合开源文档站点的需求。您可以阅读Redux迁移至Docusaurus的[讨论记录](https://github.com/reduxjs/redux/issues/3161)了解详情。

当前GitBook仅对开源和非盈利团队免费，而Docusaurus对所有人免费开放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll)是最成熟的静态站点生成器之一——事实上在Docusaurus出现前，Facebook多数开源项目网站都基于Jekyll构建！其入门极其简单，我们希望能延续这种与Jekyll相似的开发者体验。

与使用静态生成HTML并通过`<script />`标签添加交互性的方式相比，Docusaurus站点是React应用。借助现代JavaScript生态工具链，我们期望为文档站点在性能表现、资源构建流程与优化、以及易用性方面树立新标准。
