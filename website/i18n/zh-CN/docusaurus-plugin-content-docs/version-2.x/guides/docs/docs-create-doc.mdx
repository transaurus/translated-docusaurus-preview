# 创建文档

创建一个Markdown文件`greeting.md`，并将其放置在`docs`目录下。

```bash
website # root directory of your site
├── docs
│   └── greeting.md
├── src
│   └── pages
├── docusaurus.config.js
├── ...
```

```md
---
description: Create a doc page with rich content.
---

# Hello from Docusaurus

Are you ready to create the documentation site for your open source project?

## Headers

will show up on the table of contents on the upper right

So that your users will know what this page is all about without scrolling down or even without reading too much.

## Only h2 and h3 will be in the TOC by default.

You can configure the TOC heading levels either per-document or in the theme configuration.

The headers are well-spaced so that the hierarchy is clear.

- lists will help you
- present the key points
- that you want your users to remember
  - and you may nest them
    - multiple times
```

:::note

`docs`目录下所有以下划线(`_`)开头的文件默认被视为"局部"页面，将被忽略。

了解更多关于[导入局部页面的信息](../markdown-features/markdown-features-react.mdx#importing-markdown)。

:::

## 文档Front Matter {#doc-front-matter}

[Front Matter](../markdown-features/markdown-features-intro.mdx#front-matter)用于为文档页面提供额外元数据。Front Matter是可选的——Docusaurus无需Front Matter也能推断所有必要元数据。例如下文介绍的[文档标签](#doc-tags)功能就需要使用Front Matter。所有可用字段请参阅[API文档](../../api/plugins/plugin-content-docs.mdx#markdown-front-matter)。

## 文档标签 {#doc-tags}

您可以选择为文档页面添加标签，这是在[文档侧边栏](./sidebar/index.mdx)之外引入的另一种分类维度。标签以标签列表的形式在Front Matter中传递：

```md "your-doc-page.md"
---
id: doc-with-tags
title: A doc with tags
tags:
  - Demo
  - Getting started
---
```

:::tip

标签也可以使用`tags: [Demo, Getting started]`的形式声明。

了解更多[Yaml数组语法](https://www.w3schools.io/file/yaml-arrays/)。

:::

## 组织文件夹结构 {#organizing-folder-structure}

`docs`文件夹下的Markdown文件排列方式会对Docusaurus内容生成产生多重影响，但大多数影响可以与文件结构解耦。

### 文档ID {#document-id}

每个文档都有唯一的`id`。默认情况下，文档`id`是相对于docs根目录的文档名称（不含扩展名）。

例如，`greeting.md`的ID是`greeting`，而`guide/hello.md`的ID是`guide/hello`。

```bash
website # Root directory of your site
└── docs
   ├── greeting.md
   └── guide
      └── hello.md
```

但用户可以在Front Matter中自定义`id`的**最后部分**。例如若`guide/hello.md`的内容如下定义，其最终`id`将是`guide/part1`。

```md
---
id: part1
---

Lorem ipsum
```

该ID用于在手动编写侧边栏时引用文档，或使用与文档相关的布局组件或钩子时。

### 文档URL {#doc-urls}

默认情况下，文档的URL路径是其相对于`docs`文件夹的文件路径。使用`slug` Front Matter可以更改文档URL。

例如，假设您的站点结构如下：

```bash
website # Root directory of your site
└── docs
    └── guide
        └── hello.md
```

默认情况下`hello.md`将通过`/docs/guide/hello`访问。您可以通过以下方式将其URL改为`/docs/bonjour`：

```md
---
slug: /bonjour
---

Lorem ipsum
```

`slug`会被追加到文档插件的`routeBasePath`后（默认为`/docs`）。参见[纯文档模式](docs-introduction.mdx#docs-only-mode)了解如何从URL中移除`/docs`部分。

:::note

可以使用以下形式：

- 绝对slug：`slug: /mySlug`、`slug: /`...
- 相对slug：`slug: mySlug`、`slug: ./../mySlug`...

:::

若希望文档可通过根路径访问（如`https://docusaurus.io/docs/`），可使用slug Front Matter：

```md
---
id: my-home-doc
slug: /
---

Lorem ipsum
```

### 侧边栏 {#sidebars}

当使用[自动生成的侧边栏](./sidebar/autogenerated.mdx)时，文件结构将决定侧边栏的结构。

我们对于文件系统组织的建议是：使你的文件系统镜像侧边栏结构（这样你就不需要手动编写`sidebars.js`文件），并使用`slug`前置元数据来自定义每个文档的URL。
