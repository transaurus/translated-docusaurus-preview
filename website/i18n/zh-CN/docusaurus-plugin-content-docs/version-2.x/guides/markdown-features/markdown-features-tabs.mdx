# 标签页

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-tabs-styles.module.css';

Docusaurus 提供 `<Tabs>` 组件，您可以通过 [MDX](./markdown-features-react.mdx) 在 Markdown 中使用：

```jsx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana 🍌
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
    <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
    <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

---

您还可以为 `Tabs` 提供 `values` 和 `defaultValue` 属性：

```jsx
<Tabs
  defaultValue="apple"
  values={[
    {label: 'Apple', value: 'apple'},
    {label: 'Orange', value: 'orange'},
    {label: 'Banana', value: 'banana'},
  ]}>
  <TabItem value="apple">This is an apple 🍎</TabItem>
  <TabItem value="orange">This is an orange 🍊</TabItem>
  <TabItem value="banana">This is a banana 🍌</TabItem>
</Tabs>
```

:::tip

默认情况下，所有标签页都会在构建过程中被预先渲染，搜索引擎可以索引隐藏的标签页。

您可以通过 `<Tabs lazy />` 仅渲染默认标签页。

:::

## 设置默认标签页 {#displaying-a-default-tab}

默认显示第一个标签页，要覆盖此行为，可以通过在某个标签项中添加 `default` 属性来指定默认标签页。您也可以设置 `Tabs` 组件的 `defaultValue` 属性为所选标签的值。例如，在上面的示例中，为 `value="apple"` 的标签设置 `default` 属性，或者为 `Tabs` 设置 `defaultValue="apple"`，都会强制默认打开“Apple”标签页。

如果为 `Tabs` 提供的 `defaultValue` 引用了不存在的值，Docusaurus 会抛出错误。如果您希望默认不显示任何标签页，请使用 `defaultValue={null}`。

## 同步标签页选择 {#syncing-tab-choices}

您可能希望同步相同类型的标签页选择。例如，您可能希望为 Windows 用户和 macOS 用户提供不同的操作指南，并且希望通过一次点击更改所有特定于操作系统的指南标签页。为此，您可以为所有相关标签页赋予相同的 `groupId` 属性。请注意，这样做会将选择持久化到 `localStorage` 中，并且所有具有相同 `groupId` 的 `<Tab>` 实例会在其中一个的值更改时自动更新。请注意，组 ID 是全局命名空间的。

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
  </Tabs>

  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
  </Tabs>
</BrowserWindow>

对于具有相同 `groupId` 的所有标签页组，可能的值不需要相同。如果一个标签页组选择了一个在另一个具有相同 `groupId` 的标签页组中不存在的值，那么缺少该值的标签页组不会更改其标签页。您可以从以下示例中看到这一点。尝试选择 Linux，上面的标签页组不会改变。

```jsx
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">
    I am Windows.
  </TabItem>
  <TabItem value="mac" label="macOS">
    I am macOS.
  </TabItem>
  <TabItem value="linux" label="Linux">
    I am Linux.
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">I am Windows.</TabItem>
    <TabItem value="mac" label="macOS">I am macOS.</TabItem>
    <TabItem value="linux" label="Linux">I am Linux.</TabItem>
  </Tabs>
</BrowserWindow>

---

具有不同组 ID 的标签页选择不会相互干扰：

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Windows in windows.</TabItem>
  <TabItem value="mac" label="macOS">macOS is macOS.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="non-mac-operating-systems">
  <TabItem value="win" label="Windows">Windows is windows.</TabItem>
  <TabItem value="unix" label="Unix">Unix is unix.</TabItem>
</Tabs>
```

## 自定义标签页 {#customizing-tabs}

您可能希望自定义某些标签页的外观。您可以通过 `className` 属性传递字符串，指定的 CSS 类将被添加到 `Tabs` 组件中：

```jsx
// highlight-next-line
<Tabs className="unique-tabs">
  <TabItem value="Apple">This is an apple 🍎</TabItem>
  <TabItem value="Orange">This is an orange 🍊</TabItem>
  <TabItem value="Banana">This is a banana 🍌</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs className="unique-tabs">
    <TabItem value="Apple">This is an apple 🍎</TabItem>
    <TabItem value="Orange">This is an orange 🍊</TabItem>
    <TabItem value="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

### 自定义标签页标题 {#customizing-tab-headings}

您还可以通过 `attributes` 字段独立自定义每个标签页标题。额外的属性可以通过 `Tabs` 中的 `values` 属性或每个 `TabItem` 的属性传递，就像声明 `label` 一样。

```jsx title="some-doc.mdx"
import styles from './styles.module.css';

<Tabs>
  <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
    This is a banana 🍌
  </TabItem>
</Tabs>
```

```css title="styles.module.css"
.red {
  color: red;
}
.red[aria-selected='true'] {
  border-bottom-color: red;
}

.orange {
  color: orange;
}
.orange[aria-selected='true'] {
  border-bottom-color: orange;
}

.yellow {
  color: yellow;
}
.yellow[aria-selected='true'] {
  border-bottom-color: yellow;
}
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
      This is an apple 🍎
    </TabItem>
    <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
      This is an orange 🍊
    </TabItem>
    <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
      This is a banana 🍌
    </TabItem>
  </Tabs>
</BrowserWindow>

:::tip

`className` 会与其他默认类名合并使用。您也可以自定义 `data-value` 字段（如 `{'data-value': 'apple'}`）配合 CSS 属性选择器实现样式控制：

```css title="styles.module.css"
li[role='tab'][data-value='apple'] {
  color: red;
}
```

:::

## 查询字符串 {#query-string}

支持将选中的标签页状态持久化到 URL 查询参数中。该功能可实现深度链接：通过分享或收藏特定标签页链接，页面加载时将自动预选对应标签。

通过 `queryString` 属性启用此功能，并指定要使用的搜索参数名称。

```tsx
// highlight-next-line
<Tabs queryString="current-os">
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString='current-os'>
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

当用户点击标签页时，URL 末尾会自动添加搜索参数，例如：`?current-os=android` 或 `?current-os=ios`。

:::tip

`queryString` 可与 `groupId` 配合使用。

为方便起见，当 `queryString` 属性设为 `true` 时，将自动回退使用 `groupId` 的值作为参数名。

```tsx
// highlight-next-line
<Tabs groupId="current-os" queryString>
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString groupId="current-os">
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

页面加载时，系统会优先从查询字符串恢复标签页选择状态（其次才会使用 `localStorage` 存储的 `groupId` 选择状态）。

:::
