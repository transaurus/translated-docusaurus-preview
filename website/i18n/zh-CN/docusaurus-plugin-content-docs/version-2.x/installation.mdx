## 系统要求 {#requirements}

- [Node.js](https://nodejs.org/en/download/)版本16.14或更高（可通过运行`node -v`查看）。您可以使用[nvm](https://github.com/nvm-sh/nvm)管理单台机器上的多个Node版本。
  - 安装Node.js时，建议勾选所有与依赖项相关的复选框。

## 创建项目网站骨架 {#scaffold-project-website}

安装Docusaurus最简单的方法是使用命令行工具，它能帮助您快速生成一个Docusaurus网站基础框架。您可以在新的空仓库或现有仓库中的任何位置运行此命令，它将创建一个包含生成文件的新目录。

```bash
npx create-docusaurus@latest my-website classic
```

我们推荐使用`classic`模板以便快速上手，该模板包含Docusaurus 1中的功能。`classic`模板包含`@docusaurus/preset-classic`，其中集成了标准文档、博客、自定义页面和CSS框架（支持暗黑模式）。通过经典模板，您可以极速启动项目，待熟悉Docusaurus后再进行定制。

您还可以通过传递`--typescript`标志使用模板的TypeScript变体。更多信息请参阅[TypeScript支持](./typescript-support.mdx)。

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info[FB-Only]

如果您是为Facebook开源项目搭建新的Docusaurus网站，请改用`facebook`模板，该模板提供了一些有用的Facebook特定默认设置：

```bash
npx create-docusaurus@latest my-website facebook
```

:::

<details>
  <summary>Alternative installation commands</summary>

You can also initialize a new project using your preferred project manager:

```bash npm2yarn
npm init docusaurus
```

</details>

运行`npx create-docusaurus@latest --help`，或查看其[API文档](./api/misc/create-docusaurus.mdx)了解所有可用标志的详细信息。

## 项目结构 {#project-structure}

假设您选择了经典模板并将网站命名为`my-website`，您将在新目录`my-website/`下看到以下生成的文件：

```bash
my-website
├── blog
│   ├── 2019-05-28-hola.md
│   ├── 2019-05-29-hello-world.md
│   └── 2020-05-30-welcome.md
├── docs
│   ├── doc1.md
│   ├── doc2.md
│   ├── doc3.md
│   └── mdx.md
├── src
│   ├── css
│   │   └── custom.css
│   └── pages
│       ├── styles.module.css
│       └── index.js
├── static
│   └── img
├── docusaurus.config.js
├── package.json
├── README.md
├── sidebars.js
└── yarn.lock
```

### 项目结构详解 {#project-structure-rundown}

- `/blog/` - 存放博客的 Markdown 文件。如果禁用了博客插件可以删除该目录，或通过设置 `path` 选项修改目录名称。更多细节请参阅[博客指南](blog.mdx)
- `/docs/` - 存放文档的 Markdown 文件。可通过 `sidebars.js` 自定义文档侧边栏顺序。如果禁用了文档插件可以删除该目录，或通过设置 `path` 选项修改目录名称。更多细节请参阅[文档指南](./guides/docs/docs-introduction.mdx)
- `/src/` - 非文档类文件（如页面或自定义 React 组件）。虽然不强制要求将非文档文件放在此处，但集中存放便于进行代码检查/处理
  - `/src/pages` - 该目录下的所有 JSX/TSX/MDX 文件都将转换为网站页面。更多细节请参阅[页面指南](guides/creating-pages.mdx)
- `/static/` - 静态资源目录。该目录下所有内容都会被复制到最终构建的 `build` 目录根路径
- `/docusaurus.config.js` - 包含站点配置的配置文件，相当于 Docusaurus v1 中的 `siteConfig.js`
- `/package.json` - Docusaurus 网站本质是 React 应用，可以安装使用任何 npm 包
- `/sidebars.js` - 用于指定文档侧边栏的排序规则

### 单体仓库 {#monorepos}

如果正在为现有项目编写文档，采用单体仓库（monorepo）可能是理想方案。单体仓库允许在相关项目间共享依赖项。例如：网站可以直接使用本地包展示最新功能（而无需依赖发布版本），贡献者也能在实现功能时同步更新文档。下方是单体仓库的目录结构示例：

```bash
my-monorepo
├── package-a # Another package, your actual project
│   ├── src
│   └── package.json # Package A's dependencies
├── website   # Docusaurus root
│   ├── docs
│   ├── src
│   └── package.json # Docusaurus' dependencies
├── package.json # Monorepo's shared dependencies
```

这种情况下，您应该在 `./my-monorepo` 文件夹内运行 `npx create-docusaurus` 命令。

如果使用 Netlify 或 Vercel 等托管服务，需要将站点的「基础目录」修改为 Docusaurus 根目录位置（本例中应为 `./website`）。更多忽略命令配置请参阅[部署文档](./deployment.mdx#deploying-to-netlify)。

更多关于单体仓库的内容可查阅 [Yarn 文档](https://yarnpkg.com/features/workspaces)（Yarn 并非唯一解决方案，但属于常见方案），或参考 [Docusaurus](https://github.com/facebook/docusaurus) 和 [Jest](https://github.com/facebook/jest) 的实际案例。

## 运行开发服务器 {#running-the-development-server}

要实时预览文件编辑效果，可运行本地开发服务器来托管网站并同步最新变更。

```bash npm2yarn
cd my-website
npm run start
```

默认情况下，浏览器会自动打开 [`http://localhost:3000`](http://localhost:3000) 地址。

恭喜！您已成功创建首个 Docusaurus 站点！浏览网站查看现有功能。

## 构建 {#build}

Docusaurus 是现代静态网站生成器，需要将网站构建为静态内容目录并部署到 Web 服务器才能访问。构建命令如下：

```bash npm2yarn
npm run build
```

构建内容将生成在 `/build` 目录中，可部署到任意静态文件托管服务如 [GitHub Pages](https://pages.github.com/)、[Vercel](https://vercel.com/) 或 [Netlify](https://www.netlify.com/)。更多细节请参阅[部署文档](deployment.mdx)。

## 更新 Docusaurus 版本 {#updating-your-docusaurus-version}

更新 Docusaurus 版本有多种方式。最可靠的方法是手动修改 `package.json` 中的版本号为所需版本。请注意所有 `@docusaurus/` 命名空间下的包应保持版本一致。

import UpgradeGuide from '@site/src/components/UpgradeGuide';

<UpgradeGuide />

然后在包含 `package.json` 的目录中，运行您所用包管理器的安装命令：

```bash npm2yarn
npm install
```

要验证更新是否成功，请运行：

```bash
npx docusaurus --version
```

您应该能看到正确的版本号输出。

如果您使用 Yarn，也可以执行：

```bash
yarn upgrade @docusaurus/core@latest @docusaurus/preset-classic@latest
```

:::tip

通过 [`@canary` npm 分发标签](/community/canary) 使用 Docusaurus 未发布的新功能

:::
