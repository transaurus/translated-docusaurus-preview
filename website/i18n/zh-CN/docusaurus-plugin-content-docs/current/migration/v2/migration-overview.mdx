# 概述

本文档将指导您将现有的 Docusaurus 1 站点迁移至 Docusaurus 2。

我们尽可能简化迁移流程，并提供了迁移命令行工具。

## 主要差异 {#main-differences}

Docusaurus 1 是纯粹的文档站点生成器，使用 React 作为服务端模板引擎，但不在浏览器端加载 React。

Docusaurus 2 经过彻底重构，生成的单页应用可在浏览器端充分发挥 React 的全部能力。它在保留 Docusaurus 1 核心优势（快速上手、文档版本控制、国际化支持）的同时，提供了更高的可定制性。

此外，Docusaurus 2 是**高性能静态站点生成器**，能快速构建各类内容驱动型网站（如文档站、博客、产品落地页和营销页等）。

虽然我们仍聚焦于优化文档功能，但由于 Docusaurus 2 本质上是 React 应用，现已支持构建任意类型网站。**Docusaurus 不再局限于文档网站，可适用于所有网站类型。**

## Docusaurus 1 目录结构 {#docusaurus-1-structure}

Docusaurus 1 站点通常具有以下目录结构：

```bash
├── docs
└── website
    ├── blog
    ├── core
    │   └── Footer.js
    ├── package.json
    ├── pages
    ├── sidebars.json
    ├── siteConfig.js
    └── static
```

## Docusaurus 2 目录结构 {#docusaurus-2-structure}

迁移完成后，Docusaurus 2 站点目录示例如下：

```sh
├── docs
└── website
    ├── blog
    ├── src
    │   ├── components
    │   ├── css
    │   └── pages
    ├── static
    ├── package.json
    ├── sidebars.json
    ├── docusaurus.config.js
```

:::info

本次迁移不会改变 `/docs` 文件夹位置，但 Docusaurus v2 站点通常将 `/docs` 文件夹置于 `/website` 目录下

迁移至 v2 后，您可自由调整 `/docs` 文件夹的存放位置。

:::

## 迁移流程 {#migration-process}

要完成完整的 Docusaurus 2 站点迁移，需处理以下内容：

- 依赖包
- CLI 命令
- 站点配置
- Markdown 文件
- 侧边栏文件
- 页面、组件与 CSS
- 版本化文档
- 国际化支持 🚧

## 自动化迁移流程 {#automated-migration-process}

[迁移命令行工具](./migration-automated.mdx) 将自动处理大部分迁移工作。

但部分环节难以自动化，仍需手动完成迁移。

:::note

建议先运行迁移命令行工具，再根据手动迁移指南查漏补缺。

:::

## 手动迁移流程 {#manual-migration-process}

部分迁移内容（特别是页面）无法自动化处理，需手动迁移。

[手动迁移指南](./migration-manual.mdx) 提供了完整的手动操作步骤。
