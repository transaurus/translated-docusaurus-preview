# 下一步是什么？

恭喜！你现在已经掌握了 Docusaurus 的大部分核心功能。你已经：

- [使用页面插件](./creating-pages.mdx)创建了独立的 React/Markdown 页面
- [使用文档插件](./docs/docs-introduction.mdx)创建了文档页面，包括[配置侧边栏](./docs/sidebar/index.mdx)和[版本控制](./docs/versioning.mdx)
- [使用博客插件](../blog.mdx)创建了功能完整的博客
- 尝试了[各种 Markdown 功能](./markdown-features/markdown-features-intro.mdx)，这些功能对所有内容插件都很有用
- [使用样式表](../styling-layout.mdx)或[组件替换](../swizzling.mdx)来自定义网站外观
- [在页面中添加了图片和其他静态资源](../static-assets.mdx)
- [为网站添加了搜索功能](../search.mdx)
- 了解了如何通过标准 Docusaurus API 实现[浏览器支持](../browser-support.mdx)和[SEO](../seo.mdx)
- 学习了[单个插件](../using-plugins.mdx)的安装和配置方法
- [将网站部署](../deployment.mdx)到内容托管平台
- [实现国际化](../i18n/i18n-tutorial.mdx)支持多语言

此时，你可能已经有了一个庞大的`docusaurus.config.js`文件😄 但其实你还没写多少代码！大多数功能都是通过调用封装好的 Docusaurus API 实现的。在继续探索时，你可以选择以下三条路径：

- 学习更高级的 Docusaurus 概念，这将帮助你更深入地理解 API 的工作原理
- 阅读[所有现有 API](../docusaurus-core.mdx)文档，其中很多内容在指南中尚未涵盖
- 学习[开发插件](../api/plugin-methods/README.mdx)来扩展网站功能
