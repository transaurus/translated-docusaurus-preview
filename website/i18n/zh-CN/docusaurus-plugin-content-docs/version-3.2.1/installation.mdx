## 环境要求 {#requirements}

- [Node.js](https://nodejs.org/en/download/)版本18.0或更高（可通过运行`node -v`查看）。建议使用[nvm](https://github.com/nvm-sh/nvm)管理单机多Node版本。
  - 安装Node.js时，建议勾选所有与依赖相关的复选框。

## 创建项目脚手架 {#scaffold-project-website}

最简单的安装方式是通过命令行工具生成Docusaurus网站骨架。该命令可在空仓库或现有仓库任意位置执行，将创建包含脚手架文件的新目录。

```bash
npx create-docusaurus@latest my-website classic
```

推荐使用`classic`模板快速入门，该模板包含Docusaurus 1的核心功能。`classic`模板内置`@docusaurus/preset-classic`，集成标准文档系统、博客功能、自定义页面和CSS框架（支持暗黑模式）。通过经典模板可快速搭建网站，待熟悉Docusaurus后再进行深度定制。

也可通过`--typescript`标志使用TypeScript变体模板。详见[TypeScript支持](./typescript-support.mdx)。

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info[Meta专属]

若为Meta开源项目新建Docusaurus站点，请在内部仓库运行此命令，该命令包含Meta特有的默认配置：

```bash
scarf static-docs-bootstrap
```

:::

<details>
  <summary>Alternative installation commands</summary>

You can also initialize a new project using your preferred project manager:

```bash npm2yarn
npm init docusaurus
```

</details>

运行`npx create-docusaurus@latest --help`或查阅[API文档](./api/misc/create-docusaurus.mdx)获取所有可用标志的详细信息。

## 项目结构 {#project-structure}

假设选择经典模板并将站点命名为`my-website`，将在新建的`my-website/`目录下看到以下文件：

```bash
my-website
├── blog
│   ├── 2019-05-28-hola.md
│   ├── 2019-05-29-hello-world.md
│   └── 2020-05-30-welcome.md
├── docs
│   ├── doc1.md
│   ├── doc2.md
│   ├── doc3.md
│   └── mdx.md
├── src
│   ├── css
│   │   └── custom.css
│   └── pages
│       ├── styles.module.css
│       └── index.js
├── static
│   └── img
├── docusaurus.config.js
├── package.json
├── README.md
├── sidebars.js
└── yarn.lock
```

### 项目结构详解 {#project-structure-rundown}

- `/blog/` - 存放博客的 Markdown 文件。如果禁用了博客插件可删除该目录，或通过配置 `path` 选项修改目录名称。更多细节请参阅[博客指南](blog.mdx)
- `/docs/` - 存放文档的 Markdown 文件。可通过 `sidebars.js` 自定义文档侧边栏顺序。如果禁用了文档插件可删除该目录，或通过配置 `path` 选项修改目录名称。更多细节请参阅[文档指南](./guides/docs/docs-introduction.mdx)
- `/src/` - 非文档类文件（如页面或自定义 React 组件）。虽然不强制要求集中存放，但统一管理便于后续进行代码检查或处理
  - `/src/pages` - 该目录下的 JSX/TSX/MDX 文件会自动转换为网站页面。详情见[页面指南](guides/creating-pages.mdx)
- `/static/` - 静态资源目录。该目录内容会直接复制到最终构建的 `build` 目录根路径下
- `/docusaurus.config.js` - 站点配置文件，相当于 Docusaurus v1 中的 `siteConfig.js`
- `/package.json` - Docusaurus 网站本质是 React 应用，可自由安装使用任何 npm 包
- `/sidebars.js` - 用于指定文档侧边栏的排序规则

### 单体仓库（Monorepos）{#monorepos}

若需为现有项目编写文档，采用单体仓库架构是理想方案。该架构支持在关联项目间共享依赖，例如网站可直接引用本地包来展示最新功能（而非依赖发布版本），便于贡献者在开发功能时同步更新文档。典型单体仓库目录结构示例如下：

```bash
my-monorepo
├── package-a # Another package, your actual project
│   ├── src
│   └── package.json # Package A's dependencies
├── website   # Docusaurus root
│   ├── docs
│   ├── src
│   └── package.json # Docusaurus' dependencies
├── package.json # Monorepo's shared dependencies
```

此场景下，应在 `./my-monorepo` 目录内执行 `npx create-docusaurus` 命令。

若使用 Netlify 或 Vercel 等托管服务，需将站点的 `Base directory` 设置为 Docusaurus 根目录（本例中为 `./website`）。关于忽略命令配置的更多说明，请参阅[部署文档](./deployment.mdx#deploying-to-netlify)。

更多单体仓库实践可参考 [Yarn 文档](https://yarnpkg.com/features/workspaces)（Yarn 并非唯一方案），或查看 [Docusaurus](https://github.com/facebook/docusaurus) 与 [Jest](https://github.com/facebook/jest) 的真实案例。

## 启动开发服务器 {#running-the-development-server}

要实时预览文件修改效果，可运行本地开发服务器来托管网站并同步最新变更。

```bash npm2yarn
cd my-website
npm run start
```

默认情况下，浏览器会自动打开 [`http://localhost:3000`](http://localhost:3000) 页面。

恭喜！您已成功创建首个 Docusaurus 站点！现在可以浏览站点查看默认包含的功能。

## 构建 {#build}

Docusaurus 是现代化静态网站生成器，需将网站构建为静态文件后部署至 Web 服务器才能访问。执行以下命令进行构建：

```bash npm2yarn
npm run build
```

构建产物将生成在 `/build` 目录内，可将其部署至 [GitHub Pages](https://pages.github.com/)、[Vercel](https://vercel.com/) 或 [Netlify](https://www.netlify.com/) 等静态托管服务。更多细节请参阅[部署文档](deployment.mdx)。

## 更新 Docusaurus 版本 {#updating-your-docusaurus-version}

更新 Docusaurus 版本有多种方式。最可靠的方法是手动修改 `package.json` 中的版本号为所需版本。请注意所有 `@docusaurus/` 命名空间的包应保持相同版本号。

import UpgradeGuide from '@site/src/components/UpgradeGuide';

<UpgradeGuide />

然后在包含 `package.json` 的目录中，运行您所用包管理器的安装命令：

```bash npm2yarn
npm install
```

要验证更新是否成功，请运行：

```bash
npx docusaurus --version
```

您应该能看到正确的版本号输出。

如果您使用 Yarn，也可以执行：

```bash
yarn add @docusaurus/core @docusaurus/preset-classic
```

:::tip

通过 [`@canary` npm 分发标签](/community/canary) 使用 Docusaurus 未发布的新功能

:::
