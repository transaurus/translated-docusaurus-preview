# 标签页

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-tabs-styles.module.css';

Docusaurus 提供了 `<Tabs>` 组件，借助 [MDX](./markdown-features-react.mdx) 可在 Markdown 中使用：

```jsx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana 🍌
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
    <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
    <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

---

也可以为 `Tabs` 提供 `values` 和 `defaultValue` 属性：

```jsx
<Tabs
  defaultValue="apple"
  values={[
    {label: 'Apple', value: 'apple'},
    {label: 'Orange', value: 'orange'},
    {label: 'Banana', value: 'banana'},
  ]}>
  <TabItem value="apple">This is an apple 🍎</TabItem>
  <TabItem value="orange">This is an orange 🍊</TabItem>
  <TabItem value="banana">This is a banana 🍌</TabItem>
</Tabs>
```

:::tip

默认情况下，所有标签页都会在构建过程中被预先渲染，搜索引擎可以索引隐藏的标签页。

通过 `<Tabs lazy />` 可以仅渲染默认标签页。

:::

## 设置默认标签页 {#displaying-a-default-tab}

默认显示第一个标签页，要覆盖此行为，可以通过在某个标签项上添加 `default` 来指定默认标签页。也可以设置 `Tabs` 组件的 `defaultValue` 属性为所选标签的值。例如，在上面的示例中，为 `value="apple"` 的标签设置 `default` 或为标签页设置 `defaultValue="apple"` 都会强制默认打开"Apple"标签页。

如果为 `Tabs` 提供了 `defaultValue` 但该值不存在，Docusaurus 会抛出错误。如果希望默认不显示任何标签页，请使用 `defaultValue={null}`。

## 同步标签页选择 {#syncing-tab-choices}

您可能希望同步相同类型标签页的选择。例如，您可能想为 Windows 用户和 macOS 用户提供不同的操作指南，并希望通过一次点击更改所有操作系统特定的指南标签页。为此，可以为所有相关标签页赋予相同的 `groupId` 属性。这样做会将选择持久化到 `localStorage` 中，当其中一个标签页的值发生变化时，所有具有相同 `groupId` 的 `<Tab>` 实例都会自动更新。请注意，组 ID 是全局命名空间的。

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
  </Tabs>

  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
  </Tabs>
</BrowserWindow>

对于具有相同 `groupId` 的所有标签页组，可能的值不必相同。如果一个标签页组选择的值在另一个具有相同 `groupId` 的标签页组中不存在，则该标签页组不会更改其标签页。从下面的示例中可以看到这一点。尝试选择 Linux，上面的标签页组不会改变。

```jsx
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">
    I am Windows.
  </TabItem>
  <TabItem value="mac" label="macOS">
    I am macOS.
  </TabItem>
  <TabItem value="linux" label="Linux">
    I am Linux.
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">I am Windows.</TabItem>
    <TabItem value="mac" label="macOS">I am macOS.</TabItem>
    <TabItem value="linux" label="Linux">I am Linux.</TabItem>
  </Tabs>
</BrowserWindow>

---

具有不同组 ID 的标签页选择不会相互干扰：

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Windows in windows.</TabItem>
  <TabItem value="mac" label="macOS">macOS is macOS.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="non-mac-operating-systems">
  <TabItem value="win" label="Windows">Windows is windows.</TabItem>
  <TabItem value="unix" label="Unix">Unix is unix.</TabItem>
</Tabs>
```

## 自定义标签页 {#customizing-tabs}

您可能希望自定义一组标签页的外观。可以通过 `className` 属性传递字符串，指定的 CSS 类将被添加到 `Tabs` 组件中：

```jsx
// highlight-next-line
<Tabs className="unique-tabs">
  <TabItem value="Apple">This is an apple 🍎</TabItem>
  <TabItem value="Orange">This is an orange 🍊</TabItem>
  <TabItem value="Banana">This is a banana 🍌</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs className="unique-tabs">
    <TabItem value="Apple">This is an apple 🍎</TabItem>
    <TabItem value="Orange">This is an orange 🍊</TabItem>
    <TabItem value="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

### 自定义标签页标题 {#customizing-tab-headings}

您还可以通过 `attributes` 字段独立自定义每个标签页标题。额外的属性可以通过 `Tabs` 中的 `values` 属性或每个 `TabItem` 的属性传递——与声明 `label` 的方式相同。

```jsx title="some-doc.mdx"
import styles from './styles.module.css';

<Tabs>
  <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
    This is a banana 🍌
  </TabItem>
</Tabs>
```

```css title="styles.module.css"
.red {
  color: red;
}
.red[aria-selected='true'] {
  border-bottom-color: red;
}

.orange {
  color: orange;
}
.orange[aria-selected='true'] {
  border-bottom-color: orange;
}

.yellow {
  color: yellow;
}
.yellow[aria-selected='true'] {
  border-bottom-color: yellow;
}
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
      This is an apple 🍎
    </TabItem>
    <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
      This is an orange 🍊
    </TabItem>
    <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
      This is a banana 🍌
    </TabItem>
  </Tabs>
</BrowserWindow>

:::tip

`className` 会与其他默认类名合并使用。你也可以通过自定义 `data-value` 字段（如 `{'data-value': 'apple'}`）配合 CSS 属性选择器实现样式定制：

```css title="styles.module.css"
li[role='tab'][data-value='apple'] {
  color: red;
}
```

:::

## 查询字符串 {#query-string}

支持将选中的标签页状态持久化到 URL 查询参数中。此功能允许您分享已预选特定标签页的页面链接（例如从安卓应用直接跳转到预选安卓标签页的文档页）。注意：该功能不提供锚点链接，浏览器不会自动滚动至标签页位置。

通过 `queryString` 属性启用此功能，并定义要使用的搜索参数名称。

```tsx
// highlight-next-line
<Tabs queryString="current-os">
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString='current-os'>
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

当用户点击标签页时，URL 末尾会自动添加搜索参数，例如：`?current-os=android` 或 `?current-os=ios`。

:::tip

`queryString` 可与 `groupId` 配合使用。

为方便起见，当 `queryString` 属性设为 `true` 时，系统会自动使用 `groupId` 的值作为回退参数名。

```tsx
// highlight-next-line
<Tabs groupId="current-os" queryString>
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString groupId="current-os">
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

页面加载时，系统会优先从 URL 查询字符串恢复标签页选择状态（其次才会使用 `localStorage` 中存储的 `groupId` 选择状态）。

:::
