# Markdown链接

添加页面链接有两种方式：通过**URL路径**或**文件路径**。

```md
- [URL path to another document](./installation)
- [file path to another document](./installation.mdx)
```

URL路径不会被Docusaurus处理，你可以将其视为直接渲染成`<a href="./installation">`，即它会根据页面的URL位置而非文件系统位置来解析。

如果你想引用**同一插件内**的另一个Markdown文件，可以使用目标文档的相对路径。Docusaurus的Markdown加载器会将文件路径转换为目标文件的URL路径（从而移除`.md`扩展名）。

例如，如果你在`docs/folder/doc1.md`中，想要引用`docs/folder/doc2.md`、`docs/folder/subfolder/doc3.md`和`docs/otherFolder/doc4.md`：

```md title="docs/folder/doc1.md"
I am referencing a [document](doc2.mdx).

Reference to another [document in a subfolder](subfolder/doc3.mdx).

[Relative document](../otherFolder/doc4.mdx) referencing works as well.
```

相对文件路径是相对于当前文件所在目录解析的。而绝对文件路径则是相对于**内容根目录**解析的，通常是`docs/`、`blog/`或本地化目录如`i18n/zh-Hans/plugin-content-docs/current`。

绝对文件路径也可以相对于站点目录。但请注意，以`/docs/`或`/blog/`开头的链接是**不可移植的**，因为如果你创建新的文档版本或进行本地化，就需要手动更新这些链接。

```md
You can write [links](/otherFolder/doc4.mdx) relative to the content root (`/docs/`).

You can also write [links](/docs/otherFolder/doc4.mdx) relative to the site directory, but it's not recommended.
```

使用相对_文件_路径（带`.md`扩展名）而非相对_URL_链接有以下优势：

- 链接在GitHub界面和许多Markdown编辑器中仍然有效
- 你可以自定义文件的slug而无需更新所有链接
- 移动文件时，你的编辑器可能会跟踪这些更改，并自动更新文件链接
- [版本化文档](../docs/versioning.mdx)会链接到完全相同版本的另一个文档
- 相对URL链接在更新[`trailingSlash`配置](../../api/docusaurus.config.js.mdx#trailingSlash)时很容易失效

:::warning

Markdown文件引用仅在源文件和目标文件由同一插件实例处理时才有效。这是我们Markdown处理架构的技术限制，未来会修复。如果你需要在插件之间链接文件（例如从博客文章链接到文档页面），必须使用URL链接。

:::
