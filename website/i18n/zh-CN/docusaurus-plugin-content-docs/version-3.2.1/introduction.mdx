# 简介

⚡️ Docusaurus 将助您快速搭建**精美的文档站点**。

💸 构建自定义技术栈成本高昂。不如**专注于您的内容**，只需编写 Markdown 文件。

💥 想要更多功能？使用**高级特性**如版本控制、国际化、搜索和主题定制。

💅 查看**[精选 Docusaurus 站点](/showcase?tags=favorite)**获取灵感，阅读一些**[用户评价](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**静态站点生成器**。它构建了具有快速客户端导航的**单页应用**，充分利用**React**的交互能力。它提供开箱即用的**文档功能**，但也可用于创建**任何类型的站点**（个人网站、产品、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速入门 ⏱️ {#fast-track}

通过实践，**5分钟**了解 Docusaurus！

创建一个新的 Docusaurus 站点并遵循**极简**的内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 并创建新 Docusaurus 站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

访问 [`http://localhost:3000`](http://localhost:3000) 并跟随教程。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在浏览器中体验 Docusaurus！

或在线阅读 **[5分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2+ 是对 v1 的彻底重写，采用了完全现代化的工具链。[v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0)后，我们强烈建议您**使用 Docusaurus v2+ 而非 v1**，因为 v1 已被弃用。

已有[众多用户](/showcase)在使用 Docusaurus v2+（[趋势数据](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**以下情况请使用 Docusaurus v2+：**

- :white_check_mark: 您需要现代化的 Jamstack 文档站点
- :white_check_mark: 您需要具有客户端路由的单页应用 (SPA)
- :white_check_mark: 您需要完整的 React 和 MDX 功能支持
- :white_check_mark: 您不需要支持 IE11 浏览器

**以下情况请使用 [Docusaurus v1](https://v1.docusaurus.io/)：**

- :x: 您不需要单页应用 (SPA)
- :x: 您需要支持 IE11（...真的需要吗？IE [已终止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)且不再获得官方维护）

对于现有 v1 用户希望升级到 v2+ 版本，您可以遵循我们的 [迁移指南](./migration/index.mdx)。

## 功能特性 {#features}

Docusaurus 在开发者和贡献者体验方面投入了高度关注。

- ⚛️ **基于 💚 和 React 构建**：
  - 使用 React 进行扩展和定制
  - 通过提供自己的 React 组件，完全掌控网站的浏览体验
- **可插拔**：
  - 使用基础模板快速搭建站点，再通过高级功能和插件进行扩展
  - 将插件开源与社区共享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口点，便于贡献者维护
  - 热重载支持，变更时极速增量构建
  - 基于路由的代码和数据拆分
  - 轻松部署到 GitHub Pages、Netlify、Vercel 等服务

我们的共同目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建出色的文档站点。

- 🎯 **SEO 友好**：
  - 为所有可能路径静态生成 HTML 文件
  - 页面专属 SEO 优化，让用户直接访问与问题相关的官方文档
- 📝 **基于 MDX**：
  - 在 Markdown 中嵌入 JSX 和 React 编写交互式组件
  - 通过实时编辑器共享代码，让用户立即爱上您的产品
- 🔍 **搜索功能**：全站内容可搜索
- 💾 **文档版本控制**：保持文档与项目版本同步
- 🌍 **国际化 (i18n)**：支持多语言站点翻译

Docusaurus v2+ 天生具备极致的可访问性和闪电般的速度。

- ⚡️ **极速性能**。Docusaurus v2+ 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容加载如闪电般迅速。
- 🦖 **无障碍访问**。注重可访问性设计，让所有用户都能平等使用您的站点。

## 设计原则 {#design-principles}

- **学习成本低**。Docusaurus API 简洁，易于学习和使用。即使需要编写更多代码和时间，用户仍能实现大多数功能。没有抽象比错误的抽象更好，我们不希望用户被迫绕过错误的抽象。必看演讲—[最小化 API 表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直观性**。用户在查看 Docusaurus 项目目录或添加新功能时不会感到无所适从。项目结构应直观易用，基于熟悉的开发方式构建。
- **分层架构**。我们技术栈各层（内容/主题/样式）的关注点分离清晰——抽象良好且模块化。
- **合理默认值**。为用户预设常见且流行的性能优化和配置，同时保留覆盖选项。
- **无供应商锁定**。虽然强烈推荐使用默认插件和 CSS，但用户并非强制使用。由于我们对核心基础设施（如 React Loadable 和 React Router）做了默认性能优化，因此无法替换，但高层架构可选择。Markdown 引擎、CSS 框架、CSS 方法论等架构选择完全由用户决定。

我们相信，作为开发者，了解库的工作原理能帮助我们更好地使用它。因此我们致力于解释 Docusaurus 的架构和各组件，希望用户通过阅读能更深入地理解这个工具，从而更熟练地使用它。

## 与其他工具对比 {#comparison-with-other-tools}

在所有静态站点生成器中，Docusaurus 独特地专注于文档站点，并提供大量开箱即用的功能。

我们还研究了其他主流静态站点生成器，并希望分享我们的对比见解，帮助您在众多选择中找到最适合的方案。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/)功能丰富，拥有庞大的插件生态系统，能够实现Docusaurus的所有功能。但相应地，其学习曲线更为陡峭。Gatsby作为通用型建站工具表现出色，而Docusaurus则专注于将一件事做到极致——成为编写和发布内容的最佳工具。

虽然GraphQL是Gatsby的核心技术，但构建静态网站时通常不需要其提供的灵活性。

Docusaurus v2+的许多设计灵感源自Gatsby的优点，它同样是个出色的替代方案。

[Docz](https://github.com/pedronauck/docz)是基于Gatsby的文档主题，目前功能完善度不及Docusaurus。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/)是另一款流行的混合React框架。虽然能构建文档网站，但其设计并非专为文档场景优化，实现Docusaurus开箱即用的功能需要更多开发工作。

[Nextra](https://github.com/shuding/nextra)是基于Next.js的静态站点生成器，目前功能完备性低于Docusaurus。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/)与Docusaurus高度相似——都专注于内容型网站并提供开箱即用的文档功能。区别在于技术栈：VitePress基于Vue，而Docusaurus基于React。如需Vue方案，VitePress是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/)是Python生态中与Docusaurus定位相似的静态站点生成器。

如果您不需要单页应用且不计划使用React技术栈，这是个理想选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/)提供了精美的主题方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/)易于创建文档网站，但并非静态站点生成器且对SEO不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/)曾因简洁设计被众多开源项目采用。随着其转向商业化产品定位，逐渐无法满足开源文档需求。许多项目已迁移至其他方案，可参阅[Redux迁移至Docusaurus的讨论](https://github.com/reduxjs/redux/issues/3161)。

当前GitBook仅对开源和非盈利团队免费，而Docusaurus对所有人永久免费。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll)是最成熟的静态站点生成器之一——事实上在Docusaurus之前，Facebook多数开源网站都基于Jekyll构建！其入门极其简单，我们致力于提供与之媲美的开发体验。

与使用静态生成的HTML并通过`<script />`标签添加交互性的方案相比，Docusaurus站点是React应用。借助现代JavaScript生态工具链，我们期望为文档站点在性能、资源构建流程与优化、以及易用性方面树立新标准。
