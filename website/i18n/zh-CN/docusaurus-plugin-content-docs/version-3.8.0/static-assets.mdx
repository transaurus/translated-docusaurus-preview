# 静态资源

静态资源是指直接复制到构建输出中的非代码文件，包括图像、样式表、网站图标、字体等。

默认情况下，建议将这些资源放在 `static` 文件夹中。**该目录下的所有文件** 都会被复制到生成的 `build` 文件夹的根目录中，并保持目录层级结构。例如，如果你添加了一个名为 `sun.jpg` 的文件到 static 文件夹，它将被复制到 `build/sun.jpg`。

这意味着：

- 对于站点 `baseUrl: '/'`，图像 `/static/img/docusaurus.png` 将通过 `/img/docusaurus.png` 提供服务。
- 对于站点 `baseUrl: '/subpath/'`，图像 `/static/img/docusaurus.png` 将通过 `/subpath/img/docusaurus.png` 提供服务。

你可以在 `docusaurus.config.js` 中自定义静态目录来源。例如，我们可以添加 `public` 作为另一个可能的路径：

```js title="docusaurus.config.js"
export default {
  title: 'My site',
  staticDirectories: ['public', 'static'],
  // ...
};
```

现在，`public` 和 `static` 中的所有文件都将被复制到构建输出中。

## 引用静态资源 {#referencing-your-static-asset}

### 在 JSX 中 {#in-jsx}

在 JSX 中，你可以使用绝对 URL 引用 `static` 文件夹中的资源，但这并不理想，因为更改站点 `baseUrl` 会 **破坏这些链接**。对于图像 `<img src="/img/docusaurus.png" />`，如果站点托管在 `https://example.com/test`，浏览器会尝试从 URL 根目录解析它，即 `https://example.com/img/docusaurus.png`，这将失败，因为它实际上托管在 `https://example.com/test/img/docusaurus.png`。

你可以 `import()` 或 `require()` 静态资源（推荐），或者使用 `useBaseUrl` 工具函数：两者都会为你自动添加 `baseUrl` 前缀。

示例：

```jsx title="MyComponent.js"
import DocusaurusImageUrl from '@site/static/img/docusaurus.png';

<img src={DocusaurusImageUrl} />;
```

```jsx title="MyComponent.js"
<img src={require('@site/static/img/docusaurus.png').default} />
```

```jsx title="MyComponent.js"
import useBaseUrl from '@docusaurus/useBaseUrl';

<img src={useBaseUrl('/img/docusaurus.png')} />;
```

你也可以导入 SVG 文件：它们将被转换为 React 组件。

```jsx title="MyComponent.js"
import DocusaurusLogoWithKeytar from '@site/static/img/docusaurus_keytar.svg';

<DocusaurusLogoWithKeytar title="Docusaurus Logo" className="logo" />;
```

### 在 Markdown 中 {#in-markdown}

在 Markdown 中，**使用 Markdown 语法** 编写链接或图像时，可以继续使用绝对路径，因为 Docusaurus 在解析 Markdown 时会将其视为 `require` 调用而非 URL。详见 [Markdown 静态资源](./guides/markdown-features/markdown-features-assets.mdx)。

```md
You write a link like this: [Download this document](/files/note.docx)

Docusaurus changes that to: <a href={require('static/files/note.docx')}>Download this document</a>
```

:::warning[使用 Markdown 语法]

Docusaurus 仅会解析 Markdown 语法中的链接。如果你的资源引用使用的是 JSX 标签 `<a>` / `<img>`，则不会进行任何处理。

:::

### 在 CSS 中 {#in-css}

在 CSS 中，`url()` 函数通常用于引用字体和图像等资源。要引用静态资源，请使用绝对路径：

```css
@font-face {
  font-family: 'Caroline';
  src: url('/font/Caroline.otf');
}
```

`static/font/Caroline.otf` 资源将被打包器加载。

:::warning[重要提示]

一个重要提示：**永远不要硬编码你的基础 URL！** 基础 URL 被视为实现细节，应该可以轻松更改。所有路径，即使它们看起来像 URL 片段，实际上都是文件路径。

如果你觉得 URL 片段模型更容易理解，这里有一个经验法则：

- 在编写 JSX 时，假设你有一个像 `/test/` 这样的基础 URL，这样你就不会使用像 `src="/img/thumbnail.png"` 这样的绝对 URL 路径，而是 `require` 资源。
- 在编写 Markdown 或 CSS 时，假设它是 `/`，这样你总是使用不带基础 URL 的绝对路径。

:::

## 注意事项 {#caveats}

需要注意以下几点：

- 默认情况下，`static` 文件夹中的文件不会经过后处理、哈希化或压缩。
  - 但如上文所示，我们通常能将其转换为 `require` 调用以便处理。这有利于实现激进缓存策略并提升用户体验。
- 通过硬编码绝对路径引用的缺失文件在编译时不会被检测到，会导致404错误。
- 默认情况下，GitHub Pages 会使用 [Jekyll](https://jekyllrb.com/) 处理发布文件。由于 Jekyll 会忽略以下划线 `_` 开头的文件，若使用 GitHub Pages 托管，建议在 `static` 目录中添加名为 `.nojekyll` 的空文件来禁用 Jekyll。
