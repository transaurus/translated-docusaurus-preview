# 📦 plugin-pwa

Docusaurus插件，通过[Workbox](https://developers.google.com/web/tools/workbox)添加PWA支持。该插件仅在生产构建时生成[Service Worker](https://developers.google.com/web/fundamentals/primers/service-workers)，使您的文档站点具备完整的PWA特性，包括离线访问和安装功能。

## 安装 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-pwa
```

## 配置 {#configuration}

在`./static/manifest.json`路径创建[PWA清单文件](https://web.dev/add-manifest/)。

修改`docusaurus.config.js`文件，添加最小化PWA配置，例如：

```js title="docusaurus.config.js"
export default {
  plugins: [
    [
      '@docusaurus/plugin-pwa',
      {
        debug: true,
        offlineModeActivationStrategies: [
          'appInstalled',
          'standalone',
          'queryString',
        ],
        pwaHead: [
          {
            tagName: 'link',
            rel: 'icon',
            href: '/img/docusaurus.png',
          },
          {
            tagName: 'link',
            rel: 'manifest',
            href: '/manifest.json', // your PWA manifest
          },
          {
            tagName: 'meta',
            name: 'theme-color',
            content: 'rgb(37, 194, 160)',
          },
        ],
      },
    ],
  ],
};
```

## 渐进式Web应用 {#progressive-web-app}

仅安装Service Worker并不足以使您的应用成为PWA。您至少需要包含[Web应用清单](https://developer.mozilla.org/en-US/docs/Web/Manifest)，并在`<head>`中添加正确的标签（参见[选项 > pwaHead](#pwahead)）。

部署完成后，您可以使用[Lighthouse](https://developers.google.com/web/tools/lighthouse)对站点进行审计。

有关PWA要求的完整清单，请参阅[PWA检查表](https://developers.google.com/web/progressive-web-apps/checklist)

## 离线模式（预缓存） {#offline-mode-precaching}

我们通过Service Worker预缓存技术，使用户能够离线浏览Docusaurus站点。

[workbox-precaching](https://developers.google.com/web/tools/workbox/modules/workbox-precaching)页面解释了其原理：

> Service Worker的核心特性之一是在安装阶段将文件集合保存至缓存，这通常被称为"预缓存"。
>
> 这样做的主要优势在于开发者可以完全控制缓存：决定文件何时被缓存、缓存时长，并能直接向浏览器提供缓存内容而无需网络请求，从而实现离线Web应用。
>
> Workbox通过简化API和优化资源下载流程，大幅降低了预缓存实现的复杂度。

默认情况下，当站点作为应用安装时会启用离线模式。详见`offlineModeActivationStrategies`选项说明。

站点完成预缓存后，Service Worker将为后续访问提供缓存响应。当新版本部署并伴随新Service Worker时，新Worker将开始安装并进入等待状态。在此状态下，页面会显示重新加载提示，要求用户刷新以获取新内容。在用户清除应用缓存或点击提示中的`reload`按钮前，Service Worker将持续提供旧版本内容。

:::warning

离线模式/预缓存需要提前下载站点的所有静态资源，可能消耗不必要的带宽。对于某些类型的站点，启用此功能可能并非最佳选择。

:::

## 选项 {#options}

### `debug` {#debug}

- 类型: `boolean`
- 默认值: `false`

启用调试模式：

- Workbox日志
- 额外的Docusaurus日志
- 未优化的SW文件输出
- 源码映射

### `offlineModeActivationStrategies` {#offlinemodeactivationstrategies}

- 类型: `('appInstalled' | 'mobile' | 'saveData'| 'queryString' | 'always')[]`
- 默认值: `['appInstalled', 'queryString', 'standalone']`

启用离线模式的策略：

- `appInstalled`: 对已安装站点应用的用户生效（并非100%可靠）
- `standalone`: 对以独立应用模式运行的用户生效（常见于PWA安装后）
- `queryString`: 当查询字符串包含`offlineMode=true`时生效（便于PWA调试）
- `mobile`: 对移动端用户生效（`width <= 996px`）
- `saveData`: 对`navigator.connection.saveData === true`的用户生效
- `always`: 对所有用户生效

:::warning

谨慎使用：部分用户可能不喜欢被强制使用离线模式。

:::

:::danger

无法可靠检测页面是否以PWA形式渲染。

`appinstalled`事件已[从规范中移除](https://github.com/w3c/manifest/pull/836)，且[`navigator.getInstalledRelatedApps()`](https://web.dev/get-installed-related-apps/) API仅在新版Chrome中支持，并要求manifest中声明`related_applications`。

[`standalone`策略](https://petelepage.com/blog/2019/07/is-my-pwa-installed/)是激活离线模式的良好降级方案（至少当运行已安装应用时有效）。

:::

### `injectManifestConfig` {#injectmanifestconfig}

传递给`workbox.injectManifest()`的[Workbox配置项](https://developer.chrome.com/docs/workbox/reference/workbox-build/#type-InjectManifestOptions)。用于控制哪些资源将被预缓存并支持离线访问。

- 类型: `InjectManifestOptions`
- 默认值: `{}`

```js title="docusaurus.config.js"
export default {
  plugins: [
    [
      '@docusaurus/plugin-pwa',
      {
        injectManifestConfig: {
          manifestTransforms: [
            //...
          ],
          modifyURLPrefix: {
            //...
          },
          // We already add regular static assets (HTML, images...) to be available offline
          // You can add more files according to your needs
          globPatterns: ['**/*.{pdf,docx,xlsx}'],
          // ...
        },
      },
    ],
  ],
};
```

### `pwaHead` {#pwahead}

- 类型: `({ tagName: string; [attributeName: string]: string })[]`
- 默认值: `[]`

包含`tagName`和键值对属性的对象数组，用于注入`<head>`标签。技术上可注入任意head标签，但理想情况下用于使站点符合PWA规范的标签。以下是实现完全合规所需的标签列表：

```js
export default {
  plugins: [
    [
      '@docusaurus/plugin-pwa',
      {
        pwaHead: [
          {
            tagName: 'link',
            rel: 'icon',
            href: '/img/docusaurus.png',
          },
          {
            tagName: 'link',
            rel: 'manifest',
            href: '/manifest.json',
          },
          {
            tagName: 'meta',
            name: 'theme-color',
            content: 'rgb(37, 194, 160)',
          },
          {
            tagName: 'meta',
            name: 'apple-mobile-web-app-capable',
            content: 'yes',
          },
          {
            tagName: 'meta',
            name: 'apple-mobile-web-app-status-bar-style',
            content: '#000',
          },
          {
            tagName: 'link',
            rel: 'apple-touch-icon',
            href: '/img/docusaurus.png',
          },
          {
            tagName: 'link',
            rel: 'mask-icon',
            href: '/img/docusaurus.svg',
            color: 'rgb(37, 194, 160)',
          },
          {
            tagName: 'meta',
            name: 'msapplication-TileImage',
            content: '/img/docusaurus.png',
          },
          {
            tagName: 'meta',
            name: 'msapplication-TileColor',
            content: '#000',
          },
        ],
      },
    ],
  ],
};
```

### `swCustom` {#swcustom}

- 类型: `string | undefined`
- 默认值: `undefined`

用于添加自定义Workbox规则。可在此实现服务工作者支持的所有功能，并充分利用workbox库的能力。代码会被转译，因此可使用现代ES6+语法。

例如缓存外部路由文件：

```js
import {registerRoute} from 'workbox-routing';
import {StaleWhileRevalidate} from 'workbox-strategies';

// default fn export receiving some useful params
export default function swCustom(params) {
  const {
    debug, // :boolean
    offlineMode, // :boolean
  } = params;

  // Cache responses from external resources
  registerRoute((context) => {
    return [
      /graph\.facebook\.com\/.*\/picture/,
      /netlify\.com\/img/,
      /avatars1\.githubusercontent/,
    ].some((regex) => context.url.href.match(regex));
  }, new StaleWhileRevalidate());
}
```

模块需导出`default`函数，并接收若干参数。

### `swRegister` {#swregister}

- 类型: `string | false`
- 默认值: `'docusaurus-plugin-pwa/src/registerSW.js'`

在Docusaurus应用前添加入口以实现预注册。默认的`registerSW.js`文件已满足基本注册需求。

传入`false`将完全禁用注册功能。

## 自定义重新加载弹窗 {#customizing-reload-popup}

当有新服务工作者等待安装时，会渲染 `@theme/PwaReloadPopup` 组件，并向用户建议重新加载。您可以通过[组件替换](../../swizzling.mdx)该组件并实现自定义界面。组件会接收一个 `onReload` 回调作为props，当点击"刷新"按钮时应调用此回调。这将通知服务工作者安装等待中的新版本并刷新页面。

默认主题内置了重新加载弹窗的实现，采用[Infima警告框](https://infima.dev/docs/components/alert)样式。

![重新加载过程的屏幕录制。窗口右下角出现提示框显示"有新内容可用"，点击"刷新"按钮后，页面主标题从"简介"变为"PWA :))"。](/img/pwa_reload.gif)

您的组件可以返回 `null`，但不建议这样做：用户将无法获取最新内容。
