# 简介

⚡️ Docusaurus 助您快速构建**精美的文档网站**。

💸 自建技术栈成本高昂。请**专注于内容创作**，只需编写 Markdown 文件即可。

💥 想要更多功能？可使用**高级特性**如版本控制、国际化、搜索和主题定制。

💅 查看**[精选 Docusaurus 站点](/showcase?tags=favorite)**获取灵感。

🧐 Docusaurus 是**静态站点生成器**。它构建了具有快速客户端导航的**单页应用**，充分发挥 **React** 的交互能力。除提供开箱即用的**文档功能**外，也可用于创建**各类网站**（个人网站、产品页、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速上手 ⏱️ {#fast-track}

通过实操在 **5 分钟**内了解 Docusaurus！

新建 Docusaurus 站点并跟随**极简**的内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 后创建新站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

访问 [`http://localhost:3000`](http://localhost:3000) 并跟随教程。

:::tip

立即通过 **[docusaurus.new](https://docusaurus.new)** 在浏览器中体验 Docusaurus！

或在线阅读 **[5 分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2+ 是对 v1 的彻底重构，采用现代化工具链。自 [v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 后，我们强烈建议**使用 v2+ 而非已弃用的 v1 版本**。

已有 [大量用户](/showcase) 在使用 Docusaurus v2+（[趋势数据](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**适用 Docusaurus v2+ 的场景：**

- :white_check_mark: 需要现代化的 Jamstack 文档站点
- :white_check_mark: 需要支持客户端路由的单页应用 (SPA)
- :white_check_mark: 需要完整的 React 和 MDX 功能支持
- :white_check_mark: 无需兼容 IE11 浏览器

**适用 [Docusaurus v1](https://v1.docusaurus.io/) 的场景：**

- :x: 不需要单页应用 (SPA)
- :x: 需要支持 IE11（但需注意 IE [已终止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

现有 v1 用户可参照 [迁移指南](./migration/index.mdx) 升级至 v2+。

## 功能特性 {#features}

Docusaurus 以高度关注开发者和贡献者体验为核心构建。

- ⚛️ **基于💚与React构建**：
  - 使用React进行扩展和定制
  - 通过提供自定义React组件，完全掌控网站的浏览体验
- 🔌 **插件化架构**：
  - 通过基础模板快速建站，随后使用高级功能和插件
  - 将插件开源以与社区共享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口，便于贡献者维护
  - 支持热重载，变更时增量构建速度极快
  - 基于路由的代码与数据分割
  - 轻松部署至GitHub Pages、Netlify、Vercel等服务

我们的共同目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建出色的文档站点。

- 🎯 **SEO友好**：
  - 为所有可能路径静态生成HTML文件
  - 页面级SEO优化，让用户直接访问与问题相关的官方文档
- 📝 **基于MDX**：
  - 通过嵌入Markdown的JSX和React编写交互式组件
  - 在实时编辑器中共享代码，让用户即刻体验产品魅力
- 🔍 **全站搜索**：支持全站内容检索
- 💾 **文档版本控制**：保持文档与项目版本同步
- 🌍 **国际化(i18n)**：支持多语言站点翻译

Docusaurus v2+ 天生具备卓越的访问速度和普适性。

- ⚡️ **极速加载**。遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容瞬时加载
- 🦖 **无障碍访问**。注重可访问性设计，让所有用户平等获取信息

## 设计原则 {#design-principles}

- **低学习成本**：Docusaurus API简洁易学。即使需要编写更多代码，用户也能实现大多数功能。宁可缺少抽象，也不提供错误抽象——推荐观看[最小API表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)演讲
- **符合直觉**：项目目录结构直观清晰，新增功能时用户不会感到困惑，能基于熟悉的方式轻松扩展
- **分层架构**：内容/主题/样式各层关注点分离，抽象合理且模块化
- **明智默认值**：自动实施常见性能优化配置，同时保留覆盖选项
- **无供应商锁定**：鼓励但不强制使用默认插件或CSS。除React Loadable和React Router等核心基础设施外，Markdown引擎、CSS框架等架构选择完全开放

我们相信，了解库的工作原理能帮助开发者更好地使用它。因此我们详细阐释Docusaurus的架构与组件，希望用户能深入理解工具，提升使用效率。

## 与其他工具对比 {#comparison-with-other-tools}

在所有静态站点生成器中，Docusaurus专注于文档站点，提供大量开箱即用功能。

我们研究了其他主流静态站点生成器，希望通过对比分析帮助您在众多选择中找到方向。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 功能丰富，拥有庞大的插件生态系统，能够实现 Docusaurus 的所有功能。当然，这也意味着更高的学习成本。Gatsby 擅长构建各类网站，而 Docusaurus 则专注于将一件事做到极致——成为编写和发布内容的最佳工具。

GraphQL 是 Gatsby 的核心技术，但构建静态网站时通常不需要 GraphQL 提供的灵活性。

Docusaurus v2+ 的许多设计灵感都来自 Gatsby 的优点，它是个出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基于 Gatsby 的主题，用于构建文档网站。目前其功能不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一个流行的混合式 React 框架。虽然它能帮助构建文档网站，但并非专为文档场景设计，实现 Docusaurus 开箱即用的功能需要更多工作量。

[Nextra](https://github.com/shuding/nextra) 是基于 Next.js 的静态站点生成器。目前其功能不如 Docusaurus 完善。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 与 Docusaurus 有许多相似之处——都专注于内容型网站并提供开箱即用的文档功能。区别在于 VitePress 基于 Vue，而 Docusaurus 基于 React。如需 Vue 解决方案，VitePress 是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 静态站点生成器，定位与 Docusaurus 相似。

如果您不需要单页应用且不打算使用 React，这是个好选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主题。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 便于创建文档网站，但它不是静态站点生成器，且对 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 设计简洁，曾被众多开源项目采用。随着其转向商业化产品，许多功能已不再符合开源文档站点的需求。您可以阅读 [Redux 迁移至 Docusaurus 的讨论](https://github.com/reduxjs/redux/issues/3161)。

目前 GitBook 仅对开源和非盈利团队免费，而 Docusaurus 对所有人免费。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的静态站点生成器之一——事实上在 Docusaurus 之前，Facebook 多数开源网站都基于 Jekyll 构建！它极易上手，我们希望能延续这种开发体验。

与通过 `<script />` 标签添加交互性的静态 HTML 不同，Docusaurus 站点是 React 应用。借助现代 JavaScript 生态工具链，我们希望在文档站点的性能、资源构建流程、优化和易用性方面树立新标准。
