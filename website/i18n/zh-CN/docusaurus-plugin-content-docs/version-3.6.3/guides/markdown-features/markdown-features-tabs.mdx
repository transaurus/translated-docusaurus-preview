# 标签页

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-tabs-styles.module.css';

Docusaurus 提供了 `<Tabs>` 组件，借助 [MDX](./markdown-features-react.mdx) 可在 Markdown 中使用：

```jsx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana 🍌
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
    <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
    <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

---

也可以通过 `values` 和 `defaultValue` 属性配置标签页：

```jsx
<Tabs
  defaultValue="apple"
  values={[
    {label: 'Apple', value: 'apple'},
    {label: 'Orange', value: 'orange'},
    {label: 'Banana', value: 'banana'},
  ]}>
  <TabItem value="apple">This is an apple 🍎</TabItem>
  <TabItem value="orange">This is an orange 🍊</TabItem>
  <TabItem value="banana">This is a banana 🍌</TabItem>
</Tabs>
```

:::tip

默认情况下所有标签页会在构建时预渲染，搜索引擎可以索引隐藏标签页的内容。

通过 `<Tabs lazy />` 可仅渲染默认选中的标签页。

:::

## 设置默认标签页 {#displaying-a-default-tab}

默认显示第一个标签页，可通过在某个 `<TabItem>` 上添加 `default` 属性或为 `<Tabs>` 组件设置 `defaultValue` 属性来覆盖此行为。例如上例中，为 `value="apple"` 的标签页设置 `default` 属性，或为 `<Tabs>` 设置 `defaultValue="apple"` 均可强制默认展开"Apple"标签页。

若提供的 `defaultValue` 对应不存在的标签页值，Docusaurus 会抛出错误。如需默认不展开任何标签页，请使用 `defaultValue={null}`。

## 同步标签页选择状态 {#syncing-tab-choices}

您可能希望同类型标签页的选择状态保持同步。例如针对 Windows 和 macOS 用户提供不同的操作指南时，希望点击一次即可切换所有操作系统相关的标签页。为此可为相关标签页设置相同的 `groupId` 属性。该操作会将选择状态持久化到 `localStorage` 中，同组的所有 `<Tab>` 实例会在任一标签页值变更时自动同步。注意组 ID 是全局命名空间的。

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
  </Tabs>

  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
  </Tabs>
</BrowserWindow>

同组 ID 的标签页组不要求具备相同的可选值。当某标签页组选择的值在另一同组标签页中不存在时，后者将保持当前选中状态不变。如下例所示，选择 Linux 时上方标签页组不会发生变化。

```jsx
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">
    I am Windows.
  </TabItem>
  <TabItem value="mac" label="macOS">
    I am macOS.
  </TabItem>
  <TabItem value="linux" label="Linux">
    I am Linux.
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">I am Windows.</TabItem>
    <TabItem value="mac" label="macOS">I am macOS.</TabItem>
    <TabItem value="linux" label="Linux">I am Linux.</TabItem>
  </Tabs>
</BrowserWindow>

---

不同组 ID 的标签页选择互不干扰：

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Windows in windows.</TabItem>
  <TabItem value="mac" label="macOS">macOS is macOS.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="non-mac-operating-systems">
  <TabItem value="win" label="Windows">Windows is windows.</TabItem>
  <TabItem value="unix" label="Unix">Unix is unix.</TabItem>
</Tabs>
```

## 自定义标签页 {#customizing-tabs}

可通过 `className` 属性为 `<Tabs>` 组件添加 CSS 类名来自定义标签页外观：

```jsx
// highlight-next-line
<Tabs className="unique-tabs">
  <TabItem value="Apple">This is an apple 🍎</TabItem>
  <TabItem value="Orange">This is an orange 🍊</TabItem>
  <TabItem value="Banana">This is a banana 🍌</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs className="unique-tabs">
    <TabItem value="Apple">This is an apple 🍎</TabItem>
    <TabItem value="Orange">This is an orange 🍊</TabItem>
    <TabItem value="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

### 自定义标签标题 {#customizing-tab-headings}

通过 `attributes` 字段可独立定制每个标签标题。额外属性可通过 `<Tabs>` 的 `values` 属性或每个 `<TabItem>` 的 props 传递（声明方式与 `label` 属性相同）。

```jsx title="some-doc.mdx"
import styles from './styles.module.css';

<Tabs>
  <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
    This is a banana 🍌
  </TabItem>
</Tabs>
```

```css title="styles.module.css"
.red {
  color: red;
}
.red[aria-selected='true'] {
  border-bottom-color: red;
}

.orange {
  color: orange;
}
.orange[aria-selected='true'] {
  border-bottom-color: orange;
}

.yellow {
  color: yellow;
}
.yellow[aria-selected='true'] {
  border-bottom-color: yellow;
}
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
      This is an apple 🍎
    </TabItem>
    <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
      This is an orange 🍊
    </TabItem>
    <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
      This is a banana 🍌
    </TabItem>
  </Tabs>
</BrowserWindow>

:::tip

`className` 会与其他默认类名合并。你也可以使用自定义的 `data-value` 字段（如 `{'data-value': 'apple'}`）配合 CSS 属性选择器：

```css title="styles.module.css"
li[role='tab'][data-value='apple'] {
  color: red;
}
```

:::

## 查询字符串 {#query-string}

可以将选中的标签页状态持久化到 URL 的搜索参数中。这样你就能分享一个预选中特定标签页的页面链接——例如从你的 Android 应用直接链接到已预选 Android 标签页的文档。注意此功能不提供锚点链接，浏览器不会自动滚动到标签页位置。

通过 `queryString` 属性启用此功能，并定义要使用的搜索参数名称。

```tsx
// highlight-next-line
<Tabs queryString="current-os">
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString='current-os'>
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

当点击标签页时，URL 末尾会添加搜索参数：例如 `?current-os=android` 或 `?current-os=ios`。

:::tip

`queryString` 可与 `groupId` 配合使用。

为方便起见，当 `queryString` 属性设为 `true` 时，将自动使用 `groupId` 的值作为回退参数名。

```tsx
// highlight-next-line
<Tabs groupId="current-os" queryString>
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString groupId="current-os">
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

页面加载时，会优先从查询字符串恢复标签页选择状态（其次才使用 `localStorage` 存储的 `groupId` 选择状态）。

:::
