# 头部元数据

## 自定义头部元数据 {#customizing-head-metadata}

Docusaurus 会自动为您设置 `<html>`、`<head>` 和 `<body>` 中有用的页面元数据。您可以通过 Markdown 文件中的 `<head>` 标签添加额外的元数据（或覆盖现有元数据）：

```md title="markdown-features-head-metadata.mdx"
---
id: head-metadata
title: Head Metadata
---

<!-- highlight-start -->
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
<!-- highlight-end -->

# Head Metadata

My text
```

<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>

当前 Markdown 文档中已添加此 `<head>` 声明作为演示。打开浏览器开发者工具，查看此页面的元数据如何被影响。

:::note

此功能基于 Docusaurus 的 [`<Head>`](./../../docusaurus-core.mdx#head) 组件构建。有关详细文档，请参阅 [react-helmet](https://github.com/nfl/react-helmet)。

:::

:::tip[常规 SEO 无需手动设置]

内容插件（如文档和博客）提供了 `description`、`keywords` 和 `image` 等 front matter 选项，这些选项会自动应用于 `description` 和 `og:description`，而使用 `<head>` 标签时您需要手动声明两个元数据标签。

:::

## Markdown 页面描述 {#markdown-page-description}

Markdown 页面的描述元数据可能用于比头部元数据更多的地方。例如，文档插件的[生成的分类索引页](../docs/sidebar/items.mdx#generated-index-page)会使用描述元数据来生成文档卡片。

默认情况下，描述是第一行有效内容，并会尝试将其转换为纯文本。例如，以下文件...

```md
# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```

...将默认描述为 "主要内容...可能包含链接或强调"。但**此功能并非设计为完全可靠**。当无法生成合理描述时，您可以通过 front matter 显式提供：

```md
---
description: This description will override the default.
---

# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```
