# 国际化 - 使用 Git

一种**可行的翻译策略**是使用 Git（或任何其他[版本控制系统](https://en.wikipedia.org/wiki/Version_control)）**对翻译文件进行版本控制**。

## 权衡利弊 {#tradeoffs}

该策略具有以下优势：

- **易于上手**：只需将 `i18n` 文件夹提交到 Git
- **对开发者友好**：Git、GitHub 和拉取请求是主流的开发者工具
- **免费**（或无需额外成本，假设您已在使用 Git）
- **低门槛**：无需注册外部工具
- **有成就感**：贡献者会因拥有良好的贡献记录而感到高兴

使用 Git 也存在一些不足：

- **对非开发者困难**：他们不精通 Git 和拉取请求
- **对专业翻译人员困难**：他们习惯使用 SaaS 翻译软件和高级功能
- **维护困难**：必须保持翻译文件与未翻译文件**同步**

:::note

一些**大型技术项目**（如 React、Vue.js、MDN、TypeScript、Nuxt.js 等）使用 Git 进行翻译。

有关我们的研究和相关系统链接，请参阅 [Docusaurus 国际化 RFC](https://github.com/facebook/docusaurus/issues/3317)。

:::

## 初始化 {#initialization}

以下是通过 Git 将新初始化的英文 Docusaurus 网站翻译成法文的步骤，假设您已按照[国际化教程](./i18n-tutorial.mdx)操作。

### 准备 Docusaurus 站点 {#prepare-the-docusaurus-site}

初始化一个新的 Docusaurus 站点：

```bash
npx create-docusaurus@latest website classic
```

为法语添加站点配置：

```js title="docusaurus.config.js"
export default {
  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'fr'],
  },
  themeConfig: {
    navbar: {
      items: [
        // ...
        {
          type: 'localeDropdown',
          position: 'left',
        },
        // ...
      ],
    },
  },
  // ...
};
```

翻译首页：

```jsx title="src/pages/index.js"
import React from 'react';
import Translate from '@docusaurus/Translate';
import Layout from '@theme/Layout';

export default function Home() {
  return (
    <Layout>
      <h1 style={{margin: 20}}>
        <Translate description="The homepage main heading">
          Welcome to my Docusaurus translated site!
        </Translate>
      </h1>
    </Layout>
  );
}
```

### 初始化 `i18n` 文件夹 {#initialize-the-i18n-folder}

使用 [write-translations](../cli.mdx#docusaurus-write-translations-sitedir) CLI 命令初始化法语环境的 JSON 翻译文件：

```bash npm2yarn
npm run write-translations -- --locale fr

  1 translations written at i18n/fr/code.json
 11 translations written at i18n/fr/docusaurus-theme-classic/footer.json
  4 translations written at i18n/fr/docusaurus-theme-classic/navbar.json
  3 translations written at i18n/fr/docusaurus-plugin-content-docs/current.json
```

:::tip

使用 `--messagePrefix '(fr) '` 选项突出显示未翻译的字符串。

`Hello` 将显示为 `(fr) Hello`，明确表示需要翻译。

:::

将未翻译的 Markdown 文件复制到法语文件夹：

```bash
mkdir -p i18n/fr/docusaurus-plugin-content-docs/current
cp -r docs/** i18n/fr/docusaurus-plugin-content-docs/current

mkdir -p i18n/fr/docusaurus-plugin-content-blog
cp -r blog/** i18n/fr/docusaurus-plugin-content-blog

mkdir -p i18n/fr/docusaurus-plugin-content-pages
cp -r src/pages/**.md i18n/fr/docusaurus-plugin-content-pages
cp -r src/pages/**.mdx i18n/fr/docusaurus-plugin-content-pages
```

将所有文件添加到 Git。

### 翻译文件 {#translate-the-files}

翻译 `i18n/fr` 中的 Markdown 和 JSON 文件并提交翻译。

您现在应该可以启动法语站点并查看翻译：

```bash npm2yarn
npm run start -- --locale fr
```

您还可以在本地或 CI 上构建站点：

```bash npm2yarn
npm run build
# or
npm run build -- --locale fr
```

## 维护 {#maintenance}

保持翻译文件与原文**同步可能具有挑战性**，特别是对于 Markdown 文档。

### Markdown 翻译 {#markdown-translations}

当未翻译的 Markdown 文档被编辑时，**您有责任维护相应的翻译文件**，遗憾的是我们无法提供有效的协助方法。

为了保持翻译站点的一致性，当 `website/docs/doc1.md` 文档被编辑时，您需要**将这些修改反向移植**到 `i18n/fr/docusaurus-plugin-content-docs/current/doc1.md` 中。

### JSON 翻译文件维护 {#json-translations}

为协助维护 JSON 翻译文件，您可以重新运行 [write-translations](../cli.mdx#docusaurus-write-translations-sitedir) CLI 命令：

```bash npm2yarn
npm run write-translations -- --locale fr
```

新的翻译内容会被追加，现有翻译不会被覆盖。

:::tip

使用 `--override` 选项可重置您的翻译内容。

:::

### 本地化编辑链接 {#localize-edit-urls}

当用户浏览 `/fr/doc1` 页面时，编辑按钮默认会跳转到未本地化的文档 `website/docs/doc1.md`。

若您的翻译文件托管在 Git 上，可通过文档和博客插件的 `editLocalizedFiles: true` 配置项。

此时编辑按钮将指向本地化文档路径 `i18n/fr/docusaurus-plugin-content-docs/current/doc1.md`。
