# 简介

⚡️ Docusaurus 助您快速搭建**美观的文档站点**。

💸 构建自定义技术栈成本高昂。请专注于您的内容，只需编写 Markdown 文件即可。

💥 想要更多功能？使用版本控制、国际化、搜索和主题定制等**高级特性**。

💅 查看**[精选 Docusaurus 站点](/showcase?tags=favorite)**获取灵感，并阅读一些**[用户评价](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是一个**静态站点生成器**。它构建了具有快速客户端导航的**单页应用**，充分利用 **React** 的交互能力。它提供开箱即用的**文档功能**，但也可用于创建**任何类型的站点**（个人网站、产品、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速入门 ⏱️ {#fast-track}

通过实践，**5分钟**了解 Docusaurus！

创建一个新的 Docusaurus 站点并遵循**非常简短**的内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 并创建新的 Docusaurus 站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

访问 [`http://localhost:3000`](http://localhost:3000) 并跟随教程操作。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在浏览器中体验 Docusaurus！

或在线阅读 **[5分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2+ 是对 v1 的彻底重写，采用了完全现代化的工具链。[v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 后，我们强烈建议您**使用 Docusaurus v2+ 而非 v1**，因为 v1 已停止维护。

已有[大量用户](/showcase)正在使用 Docusaurus v2+（[趋势数据](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**适合使用 Docusaurus v2+ 的场景：**

- :white_check_mark: 需要现代化的 Jamstack 文档站点
- :white_check_mark: 需要具有客户端路由的单页应用 (SPA)
- :white_check_mark: 需要完整的 React 和 MDX 功能
- :white_check_mark: 不需要支持 IE11

**适合使用 [Docusaurus v1](https://v1.docusaurus.io/) 的场景：**

- :x: 不需要单页应用 (SPA)
- :x: 需要支持 IE11（...真的需要吗？IE [已终止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11) 且不再获得官方维护）

对于现有v1用户希望升级至v2+版本，可遵循我们的[迁移指南](./migration/index.mdx)。

## 功能特性 {#features}

Docusaurus在开发者和贡献者体验方面倾注了高度关注。

- ⚛️ **基于💚与React构建**：
  - 使用React进行扩展和定制
  - 通过提供自定义React组件完全掌控站点浏览体验
- **插件化架构**：
  - 通过基础模板快速建站，随后使用高级功能和插件
  - 将插件开源以与社区共享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口点，便于贡献者维护
  - 支持热重载，变更时增量构建速度极快
  - 基于路由的代码与数据分割
  - 轻松部署至GitHub Pages、Netlify、Vercel等服务

我们共同的目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建出色的文档站点。

- 🎯 **SEO友好**：
  - 为所有可能路径静态生成HTML文件
  - 页面专属SEO优化，让用户直接访问与问题相关的官方文档
- 📝 **基于MDX驱动**：
  - 在Markdown中嵌入JSX和React编写交互式组件
  - 通过实时编辑器共享代码，让用户即刻爱上您的产品
- 🔍 **全站搜索**：支持全站内容检索
- 💾 **文档版本控制**：保持文档与项目版本同步
- 🌍 **国际化(i18n)**：支持多语言站点翻译

Docusaurus v2+天生具备极致的访问速度和普适的可访问性。

- ⚡️ **闪电般快速**。遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容极速加载
- 🦖 **无障碍设计**。注重可访问性，让所有用户平等获取信息

## 设计原则 {#design-principles}

- **低学习成本**。Docusaurus应易于学习和使用，因其API非常精简。即使需要编写更多代码和时间，用户仍可实现大多数功能。没有抽象好过错误的抽象，我们不希望用户被迫绕过错误抽象。必看演讲——《[最小化API表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)》
- **符合直觉**。用户在查看Docusaurus项目目录或添加新功能时不会感到无所适从。它应该直观易用，能基于熟悉的方式轻松扩展
- **分层架构**。各层级（内容/主题/样式）关注点分离明确——抽象良好且模块化
- **合理默认值**。为常见场景默认实施性能优化和配置，同时允许覆盖
- **无供应商锁定**。不强制使用默认插件或CSS（但强烈推荐）。核心基础设施如React Loadable和React Router不可替换（因默认进行性能优化），但高层架构如Markdown引擎、CSS框架等完全由用户选择

我们相信，作为开发者，了解库的工作原理能帮助我们更好地使用它。因此我们致力于解析Docusaurus的架构和组件，希望用户通过阅读能更深入地理解工具，从而更熟练地运用它。

## 与其他工具对比 {#comparison-with-other-tools}

在所有静态站点生成器中，Docusaurus独树一帜地专注于文档站点，并提供大量开箱即用的功能。

我们还研究了其他主流静态站点生成器，并希望分享我们的对比见解，帮助您在众多选择中找到最适合的方案。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/)功能丰富，拥有庞大的插件生态系统，能够实现Docusaurus的所有功能。但相应地，其学习曲线更为陡峭。Gatsby擅长构建各类网站，而Docusaurus则专注于将一件事做到极致——成为编写和发布内容的最佳工具。

GraphQL是Gatsby的核心技术，但构建静态网站时通常不需要GraphQL提供的灵活性。

Docusaurus v2+的许多设计灵感来源于Gatsby的优点，它也是一个优秀的替代方案。

[Docz](https://github.com/pedronauck/docz)是基于Gatsby的文档网站主题，目前功能不如Docusaurus完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/)是另一个流行的混合React框架。虽然可以用于构建文档网站，但它并非专为此场景设计，实现Docusaurus开箱即用的功能需要更多工作量。

[Nextra](https://github.com/shuding/nextra)是基于Next.js的静态站点生成器，目前功能不如Docusaurus全面。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/)与Docusaurus有许多相似之处——都专注于内容型网站并提供开箱即用的文档功能。区别在于VuePress基于Vue，而Docusaurus基于React。如需Vue解决方案，VuePress是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/)是流行的Python静态站点生成器，其价值主张与Docusaurus相似。

如果您不需要单页应用且不打算使用React，这是个不错的选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/)是个精美的主题。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/)易于创建文档网站，但它不是静态站点生成器，也不利于SEO优化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/)设计简洁，曾被许多开源项目使用。但随着其转向商业化产品，已不再符合开源项目文档需求。许多项目已转向其他方案，如Redux迁移至Docusaurus的[案例](https://github.com/reduxjs/redux/issues/3161)。

目前GitBook仅对开源和非盈利团队免费，而Docusaurus对所有人免费。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll)是最成熟的静态站点生成器之一——事实上在Docusaurus之前，Facebook多数开源网站都基于Jekyll构建！它极易上手，我们希望能提供类似Jekyll的开发体验。

与使用静态生成的HTML并通过`<script />`标签添加交互性的方案相比，Docusaurus站点本质上是React应用。借助现代JavaScript生态工具链，我们致力于为文档站点设立性能、资源构建流程与优化、以及易用性方面的新标准。

新功能请求请发布到我们的[功能需求看板(Canny)](/feature-requests)。该工具便于路线规划，并支持按投票数排序，相比难以分类的GitHub issues，能更直观地反映高优先级需求。请注意避免直接提交大型新功能的Pull Request——可能已有成员在开发或该功能已列入路线图，请先与我们沟通！
