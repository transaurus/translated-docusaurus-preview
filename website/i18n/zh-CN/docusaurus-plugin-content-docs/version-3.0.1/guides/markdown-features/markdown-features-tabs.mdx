# 标签页

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-tabs-styles.module.css';

Docusaurus 提供 `<Tabs>` 组件，通过 [MDX](./markdown-features-react.mdx) 可在 Markdown 中使用：

```jsx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana 🍌
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
    <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
    <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

---

也可以为 `Tabs` 提供 `values` 和 `defaultValue` 属性：

```jsx
<Tabs
  defaultValue="apple"
  values={[
    {label: 'Apple', value: 'apple'},
    {label: 'Orange', value: 'orange'},
    {label: 'Banana', value: 'banana'},
  ]}>
  <TabItem value="apple">This is an apple 🍎</TabItem>
  <TabItem value="orange">This is an orange 🍊</TabItem>
  <TabItem value="banana">This is a banana 🍌</TabItem>
</Tabs>
```

:::tip

默认情况下，所有标签页都会在构建时预渲染，搜索引擎可以索引隐藏的标签页。

使用 `<Tabs lazy />` 可仅渲染默认标签页。

:::

## 设置默认标签页 {#displaying-a-default-tab}

默认显示第一个标签页，要覆盖此行为，可通过在某个标签项添加 `default` 属性来指定默认标签页。也可通过设置 `Tabs` 组件的 `defaultValue` 属性为指定标签值。例如上例中，为 `value="apple"` 标签设置 `default` 属性，或在 `Tabs` 上设置 `defaultValue="apple"`，都会强制默认展开"Apple"标签页。

若为 `Tabs` 提供的 `defaultValue` 指向不存在的值，Docusaurus 会抛出错误。如需默认不显示任何标签页，请使用 `defaultValue={null}`。

## 同步标签页选择 {#syncing-tab-choices}

您可能希望同类型标签页的选择能相互同步。例如，您可能想为 Windows 用户和 macOS 用户提供不同的操作指南，并希望通过一次点击切换所有操作系统相关的标签页。为此，可为所有相关标签页赋予相同的 `groupId` 属性。注意：此操作会将选择状态持久化到 `localStorage` 中，且所有具有相同 `groupId` 的 `<Tab>` 实例会在任一实例值变更时自动更新。需注意组 ID 是全局命名空间的。

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
  </Tabs>

  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
    <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
  </Tabs>
</BrowserWindow>

对于具有相同 `groupId` 的所有标签页组，其可选值不必相同。如果某标签页组选择的值在另一个同组 ID 的标签页组中不存在，则该标签页组不会切换标签页。从下例可见：选择 Linux 时，上方标签页组不会变化。

```jsx
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">
    I am Windows.
  </TabItem>
  <TabItem value="mac" label="macOS">
    I am macOS.
  </TabItem>
  <TabItem value="linux" label="Linux">
    I am Linux.
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs groupId="operating-systems">
    <TabItem value="win" label="Windows">I am Windows.</TabItem>
    <TabItem value="mac" label="macOS">I am macOS.</TabItem>
    <TabItem value="linux" label="Linux">I am Linux.</TabItem>
  </Tabs>
</BrowserWindow>

---

不同组 ID 的标签页选择互不干扰：

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Windows in windows.</TabItem>
  <TabItem value="mac" label="macOS">macOS is macOS.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="non-mac-operating-systems">
  <TabItem value="win" label="Windows">Windows is windows.</TabItem>
  <TabItem value="unix" label="Unix">Unix is unix.</TabItem>
</Tabs>
```

## 自定义标签页 {#customizing-tabs}

您可能需要自定义某组标签页的外观。可通过 `className` 属性传入字符串，指定的 CSS 类会被添加到 `Tabs` 组件：

```jsx
// highlight-next-line
<Tabs className="unique-tabs">
  <TabItem value="Apple">This is an apple 🍎</TabItem>
  <TabItem value="Orange">This is an orange 🍊</TabItem>
  <TabItem value="Banana">This is a banana 🍌</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs className="unique-tabs">
    <TabItem value="Apple">This is an apple 🍎</TabItem>
    <TabItem value="Orange">This is an orange 🍊</TabItem>
    <TabItem value="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

### 自定义标签页标题 {#customizing-tab-headings}

您还可以通过 `attributes` 字段独立自定义每个标签页标题。额外属性可通过 `Tabs` 的 `values` 属性传递，或像声明 `label` 那样通过每个 `TabItem` 的 props 传递。

```jsx title="some-doc.mdx"
import styles from './styles.module.css';

<Tabs>
  <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
    This is a banana 🍌
  </TabItem>
</Tabs>
```

```css title="styles.module.css"
.red {
  color: red;
}
.red[aria-selected='true'] {
  border-bottom-color: red;
}

.orange {
  color: orange;
}
.orange[aria-selected='true'] {
  border-bottom-color: orange;
}

.yellow {
  color: yellow;
}
.yellow[aria-selected='true'] {
  border-bottom-color: yellow;
}
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
      This is an apple 🍎
    </TabItem>
    <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
      This is an orange 🍊
    </TabItem>
    <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
      This is a banana 🍌
    </TabItem>
  </Tabs>
</BrowserWindow>

:::tip

`className` 会与其他默认类名合并。你也可以使用自定义的 `data-value` 字段（如 `{'data-value': 'apple'}`）配合 CSS 属性选择器：

```css title="styles.module.css"
li[role='tab'][data-value='apple'] {
  color: red;
}
```

:::

## 查询字符串 {#query-string}

可以将选中的标签页状态持久化到 URL 的搜索参数中。这样您就能分享一个预选中特定标签页的页面链接——例如从您的 Android 应用直接跳转到已预选 Android 标签页的文档页面。此功能不提供锚点链接，浏览器不会自动滚动到标签页位置。

通过 `queryString` 属性启用此功能，并定义要使用的搜索参数名称。

```tsx
// highlight-next-line
<Tabs queryString="current-os">
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString='current-os'>
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

当点击标签页时，URL 末尾会添加搜索参数：`?current-os=android` 或 `?current-os=ios`。

:::tip

`queryString` 可与 `groupId` 配合使用。

为方便起见，当 `queryString` 属性设为 `true` 时，将自动回退使用 `groupId` 的值作为参数名。

```tsx
// highlight-next-line
<Tabs groupId="current-os" queryString>
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString groupId="current-os">
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

页面加载时，会优先从 URL 查询字符串恢复标签页选择（而非使用 `localStorage` 存储的 `groupId` 选择状态）。

:::
