# 简介

⚡️ Docusaurus 将帮助您**快速部署精美的文档网站**。

💸 构建自定义技术栈成本高昂。**专注于您的内容**，只需编写 Markdown 文件即可。

💥 想要更多功能？使用**高级特性**如版本控制、国际化、搜索和主题定制。

💅 查看**[最佳 Docusaurus 网站](/showcase?tags=favorite)**获取灵感。

🧐 Docusaurus 是一个**静态网站生成器**。它构建了一个具有快速客户端导航的**单页应用程序**，充分利用 **React** 的威力使您的网站具有交互性。它提供了开箱即用的**文档功能**，但也可用于创建**任何类型的网站**（个人网站、产品、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速入门 ⏱️ {#fast-track}

通过实际操作，**5 分钟**了解 Docusaurus！

创建一个新的 Docusaurus 站点并遵循**非常简短**的内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 并创建一个新的 Docusaurus 站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

打开 [`http://localhost:3000`](http://localhost:3000) 并按照教程操作。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在浏览器中测试 Docusaurus！

或在线阅读 **[5 分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2+ 是对 Docusaurus v1 的完全重写，利用了完全现代化的工具链。在 [v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 后，我们强烈建议您**使用 Docusaurus v2+ 而非 Docusaurus v1**，因为 Docusaurus v1 已被弃用。

[许多用户](/showcase) 已经在使用 Docusaurus v2+（[趋势](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**在以下情况下使用 Docusaurus v2+：**

- :white_check_mark: 您想要一个现代化的 Jamstack 文档网站
- :white_check_mark: 您想要一个具有客户端路由的单页应用程序 (SPA)
- :white_check_mark: 您想要充分利用 React 和 MDX 的全部功能
- :white_check_mark: 您不需要支持 IE11

**在以下情况下使用 [Docusaurus v1](https://v1.docusaurus.io/)：**

- :x: 您不想要单页应用程序 (SPA)
- :x: 您需要支持 IE11（...真的需要吗？IE [已停止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11) 且不再官方支持）

对于现有 v1 用户想要升级到 v2+，您可以遵循我们的 [迁移指南](./migration/index.mdx)。

## 功能 {#features}

Docusaurus 以极高的开发者与贡献者体验为核心构建。

- ⚛️ **基于💚与React构建**：
  - 使用React进行扩展和定制
  - 通过提供自定义React组件，完全掌控站点的浏览体验
- 🔌 **插件化架构**：
  - 通过基础模板快速建站，随后使用高级功能和插件
  - 将插件开源以与社区共享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口，便于贡献者维护
  - 支持热重载，变更时增量构建速度极快
  - 基于路由的代码与数据分割
  - 轻松部署至GitHub Pages、Netlify、Vercel等服务

我们的共同目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建高效优质的文档站点。

- 🎯 **SEO友好**：
  - 为所有路径静态生成HTML文件
  - 页面专属SEO优化，让用户直接访问与问题相关的官方文档
- 📝 **基于MDX驱动**：
  - 在Markdown中嵌入JSX和React编写交互式组件
  - 通过实时编辑器共享代码，让用户即刻爱上您的产品
- 🔍 **全站搜索**：支持全站内容检索
- 💾 **文档版本控制**：保持文档与项目版本同步
- 🌍 **国际化(i18n)**：支持多语言站点翻译

Docusaurus v2+ 天生具备极致的访问速度，并致力于为所有用户提供无障碍访问体验。

- ⚡️ **闪电般快速**。遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容瞬时加载
- 🦖 **无障碍设计**。注重可访问性，让所有用户平等获取信息

## 设计原则 {#design-principles}

- **低学习成本**。Docusaurus API简洁易用，即使需要编写更多代码，用户也能实现大多数功能。宁可缺少抽象也不提供错误抽象，避免用户被迫绕过不当设计。推荐观看演讲——《[最小化API表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)》
- **符合直觉**。项目目录结构直观清晰，新增功能时用户不会感到无所适从，基于熟悉的方式即可轻松扩展
- **分层架构**。内容层/主题层/样式层关注点分离，抽象合理且模块化
- **明智的默认值**。自动实施常见性能优化与配置，同时保留覆盖选项
- **无供应商锁定**。虽强烈推荐默认插件和CSS，但均可替换。仅核心基础设施（如React Loadable和React Router）因默认性能优化不可替换，其他如Markdown引擎、CSS框架等架构选择完全开放

我们相信，了解库的工作原理能帮助开发者更好地使用它。因此我们详细阐释Docusaurus的架构与组件，希望用户能深入理解工具，从而更高效地运用。

## 与其他工具的对比 {#comparison-with-other-tools}

在所有静态站点生成器中，Docusaurus独树一帜地专注于文档站点，并提供大量开箱即用功能。

我们深入研究了其他主流静态站点生成器，希望通过对比分析帮助您在众多选择中找到方向。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 功能丰富，拥有庞大的插件生态系统，能够实现 Docusaurus 的所有功能。当然，这也意味着更高的学习成本。Gatsby 擅长处理多种场景，适合构建各类网站。而 Docusaurus 则专注于将一件事做到极致——成为编写和发布内容的最佳工具。

GraphQL 是 Gatsby 的核心技术，但构建静态网站时通常不需要 GraphQL 提供的灵活性。

Docusaurus v2+ 的许多设计灵感都来自 Gatsby 的优秀特性，它是个很好的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基于 Gatsby 的主题化文档工具，目前功能不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一个流行的混合式 React 框架。虽然可以用来构建文档网站，但它并非专为文档场景设计，实现 Docusaurus 开箱即用的功能需要更多开发工作。

[Nextra](https://github.com/shuding/nextra) 是基于 Next.js 的静态站点生成器，目前功能不如 Docusaurus 完善。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 与 Docusaurus 有许多相似之处——都专注于内容型网站并提供开箱即用的文档功能。区别在于 VitePress 基于 Vue，而 Docusaurus 基于 React。如需 Vue 技术栈，VitePress 是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 静态站点生成器，定位与 Docusaurus 相似。

如果您不需要单页应用且不打算使用 React，这是个好选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主题。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 易于创建文档网站，但它不是静态站点生成器且对 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 设计简洁，曾被众多开源项目采用。随着其转向商业化产品，许多功能已不再符合开源文档需求。您可以阅读 [Redux 迁移至 Docusaurus 的讨论](https://github.com/reduxjs/redux/issues/3161)了解详情。

当前 GitBook 仅对开源和非营利团队免费，而 Docusaurus 对所有人免费。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的静态站点生成器之一——事实上在 Docusaurus 之前，Facebook 多数开源网站都基于 Jekyll 构建！它的入门极其简单，我们希望能延续这种开发体验。

与传统静态 HTML 通过 `<script />` 标签添加交互的方式不同，Docusaurus 站点是 React 应用。借助现代 JavaScript 工具链，我们希望在文档站点的性能、资源构建流程、优化效果和易用性方面树立新标准。

对于新功能请求，您可以在我们的[功能需求看板(Canny)](/feature-requests)上发帖。该看板是路线规划的高效工具，支持按投票数排序，这能让核心团队更清晰地识别高需求功能（相比GitHub issues更易分类）。请注意：请勿直接为新功能（尤其是大型功能）提交Pull Request，因为可能已有成员在开发或该功能已列入路线图。请先与我们沟通！
