# 升级 Docusaurus

Docusaurus 采用 `主版本号.次版本号.修订号` 的版本控制方案，并遵循[语义化版本规范](https://semver.org/)。

**重大变更**仅会在主版本升级时发布，并详细记录在以下升级指南中。

## 升级故障排查

升级 Docusaurus 时，您可能会遇到因依赖项缓存不匹配导致的问题——在提交错误报告或寻求支持前，应先执行以下常见问题排查步骤。

### 运行 `clear` 命令

该 CLI 命令用于清除 Docusaurus 站点生成的静态资源、缓存和构建产物。

```bash npm2yarn
npm run clear
```

### 删除 `node_modules` 和锁文件

使用以下命令移除 `node_modules` 文件夹和包管理器的锁文件：

<Tabs>
<TabItem label="Bash" value="bash">

```bash
rm -rf node_modules yarn.lock package-lock.json
```

</TabItem>
<TabItem label="PowerShell" value="powershell">

```powershell
@('node_modules','yarn.lock','package-lock.json') | Remove-Item -Force -Recurse -ErrorAction SilentlyContinue
```

</TabItem>
</Tabs>

随后重新安装依赖并重新生成锁文件：

```bash npm2yarn
npm install
```
