# 📦 plugin-client-redirects

import APITable from '@site/src/components/APITable';

用于生成**客户端重定向**的Docusaurus插件。

该插件会向静态站点写入额外的HTML页面，通过JavaScript将用户重定向至现有的Docusaurus页面。

:::warning[仅生产环境生效]

本插件在开发环境中始终处于非活跃状态，**仅在生产环境生效**，因其功能基于构建输出实现。

:::

:::warning

建议优先使用服务端重定向方案。

使用本插件前，请确认您的托管服务商是否已提供此功能。

:::

## 安装 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-client-redirects
```

## 配置 {#configuration}

可配置字段：

<APITable>

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `fromExtensions` | `string[]` | `[]` | The extensions to be removed from the route after redirecting. |
| `toExtensions` | `string[]` | `[]` | The extensions to be appended to the route after redirecting. |
| `redirects` | <code>[RedirectRule](#RedirectRule)[]</code> | `[]` | The list of redirect rules. |
| `createRedirects` | <code>[CreateRedirectsFn](#CreateRedirectsFn)</code> | `undefined` | A callback to create a redirect rule. Docusaurus query this callback against every path it has created, and use its return value to output more paths. |

</APITable>

:::note

本插件会读取[`siteConfig.onDuplicateRoutes`](../docusaurus.config.js.mdx#onDuplicateRoutes)配置，当多个文件将输出至相同位置时调整其日志级别。

:::

### 类型定义 {#types}

#### `RedirectRule` {#RedirectRule}

```ts
type RedirectRule = {
  to: string;
  from: string | string[];
};
```

:::note

"from"与"to"的概念是本插件的核心。"from"表示您希望创建的路径（即待写入的额外HTML文件）；"to"表示您希望重定向至的路径（通常是Docusaurus已识别的路由）。

因此可以对同一个"to"设置多个"from"：我们将创建多个最终指向同一目标的HTML文件。但一个"from"绝不能对应多个"to"：写入的HTML文件必须具有确定的重定向目标。

:::

#### `CreateRedirectsFn` {#CreateRedirectsFn}

```ts
// The parameter `path` is a route that Docusaurus has already created. It can
// be seen as the "to", and your return value is the "from". Returning a falsy
// value will not create any redirect pages for this particular path.
type CreateRedirectsFn = (path: string) => string[] | string | null | undefined;
```

### 配置示例 {#ex-config}

配置示例如下：

```js title="docusaurus.config.js"
export default {
  plugins: [
    [
      '@docusaurus/plugin-client-redirects',
      // highlight-start
      {
        fromExtensions: ['html', 'htm'], // /myPage.html -> /myPage
        toExtensions: ['exe', 'zip'], // /myAsset -> /myAsset.zip (if latter exists)
        redirects: [
          // /docs/oldDoc -> /docs/newDoc
          {
            to: '/docs/newDoc',
            from: '/docs/oldDoc',
          },
          // Redirect from multiple old paths to the new path
          {
            to: '/docs/newDoc2',
            from: ['/docs/oldDocFrom2019', '/docs/legacyDocFrom2016'],
          },
        ],
        createRedirects(existingPath) {
          if (existingPath.includes('/community')) {
            // Redirect from /docs/team/X to /community/X and /docs/support/X to /community/X
            return [
              existingPath.replace('/community', '/docs/team'),
              existingPath.replace('/community', '/docs/support'),
            ];
          }
          return undefined; // Return a falsy value: no redirect created
        },
      },
      // highlight-end
    ],
  ],
};
```
