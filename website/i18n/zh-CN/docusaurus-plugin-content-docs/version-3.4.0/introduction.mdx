# 简介

⚡️ Docusaurus 助您快速搭建**美观的文档站点**。

💸 定制技术栈成本高昂。不如专注内容创作，**只需编写 Markdown 文件**。

💥 需要更多功能？可使用**高级特性**如版本控制、国际化、搜索和主题定制。

💅 查看**[精选 Docusaurus 站点](/showcase?tags=favorite)**获取灵感，阅读**[用户评价](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**静态站点生成器**。它构建**单页应用**，通过快速的客户端导航和完整的**React**交互能力，提供开箱即用的**文档功能**，也能创建**任意类型站点**（个人网站、产品页、博客、营销落地页等）。

![](/img/slash-introducing.svg)

## 快速上手 ⏱️ {#fast-track}

通过实践**5分钟**了解 Docusaurus！

新建 Docusaurus 站点并跟随**极简**内置教程。

安装 [Node.js](https://nodejs.org/en/download/) 后创建新站点：

```bash
npx create-docusaurus@latest my-website classic
```

启动站点：

```bash
cd my-website
npx docusaurus start
```

访问 [`http://localhost:3000`](http://localhost:3000) 跟随教程。

:::tip

立即通过 **[docusaurus.new](https://docusaurus.new)** 在线体验 Docusaurus！

或阅读在线版 **[5分钟教程](https://tutorial.docusaurus.io)**。

:::

## 从 v1 迁移 {#migrating-from-v1}

Docusaurus v2+ 是对 v1 的彻底重构，采用现代化工具链。[v2 正式发布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0)后，我们强烈建议**使用 v2+ 而非已弃用的 v1 版本**。

[众多用户](/showcase) 已在使用 v2+（[趋势数据](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**适用 v2+ 的场景：**

- :white_check_mark: 需要现代化的 Jamstack 文档站点
- :white_check_mark: 需要支持客户端路由的单页应用 (SPA)
- :white_check_mark: 需要完整的 React 和 MDX 能力
- :white_check_mark: 无需支持 IE11

**适用 [v1](https://v1.docusaurus.io/) 的场景：**

- :x: 不需要单页应用 (SPA)
- :x: 需要支持 IE11（注意：IE [已终止支持](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

对于现有v1用户希望升级到v2+版本，可遵循我们的[迁移指南](./migration/index.mdx)。

## 功能特性 {#features}

Docusaurus在开发者和贡献者体验方面倾注了高度关注。

- ⚛️ **基于💚与React构建**：
  - 使用React进行扩展和定制
  - 通过提供自定义React组件，全面掌控站点浏览体验
- **插件化架构**：
  - 通过基础模板快速建站，再使用高级功能和插件
  - 将插件开源以与社区共享
- ✂️ **开发者体验**：
  - 立即开始编写文档
  - 统一的配置入口点，便于贡献者维护
  - 变更时支持热重载的闪电级增量构建
  - 基于路由的代码与数据拆分
  - 轻松部署至GitHub Pages、Netlify、Vercel等服务

我们的共同目标——帮助用户快速找到所需内容并更好地理解您的产品。我们分享最佳实践，助您构建出色的文档站点。

- 🎯 **SEO友好**：
  - 为所有可能路径静态生成HTML文件
  - 页面专属SEO优化，让用户直接定位到相关官方文档
- 📝 **基于MDX驱动**：
  - 在Markdown中嵌入JSX和React编写交互式组件
  - 通过实时编辑器共享代码，让用户即刻爱上您的产品
- 🔍 **全站搜索**：支持全站内容检索
- 💾 **文档版本控制**：保持文档与项目版本同步
- 🌍 **国际化(i18n)**：支持多语言站点翻译

Docusaurus v2+天生具备极致的访问速度和普适的可访问性。

- ⚡️ **闪电般快速**。遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，确保内容极速加载
- 🦖 **无障碍设计**。注重可访问性，让所有用户平等获取信息

## 设计原则 {#design-principles}

- **低学习成本**。Docusaurus应易于学习和使用，因其API非常精简。即使需要编写更多代码和时间，用户仍能实现大多数功能。没有抽象优于错误的抽象，我们不希望用户被迫绕过错误抽象。必看演讲——《[最小API表面积](https://www.youtube.com/watch?v=4anAwXYqLG8)》
- **符合直觉**。用户在查看Docusaurus项目目录或添加新功能时不会感到无所适从。它应该直观易用，基于熟悉的开发方式构建
- **分层架构**。各层级（内容/主题/样式）关注点分离明确——抽象良好且模块化
- **合理默认值**。为常见场景自动实施性能优化和配置，同时保留覆盖选项
- **无供应商锁定**。不强制使用默认插件或CSS（但强烈推荐）。核心基础设施如React Loadable和React Router不可替换（因默认进行性能优化），但高层架构如Markdown引擎、CSS框架和方法论完全由用户选择

我们相信，作为开发者，了解库的工作原理能帮助我们更好地使用它。因此我们致力于解析Docusaurus的架构和组件，希望用户通过阅读获得更深理解，从而更高效地使用工具。

## 与其他工具对比 {#comparison-with-other-tools}

在所有静态站点生成器中，Docusaurus独树一帜地专注于文档站点，并提供大量开箱即用的功能。

我们还研究了其他主流静态站点生成器，并希望分享我们的对比见解，帮助您在众多选择中找到最适合的方案。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 功能丰富，拥有庞大的插件生态系统，能够实现 Docusaurus 的所有功能。但相应地，其学习曲线更为陡峭。Gatsby 是构建各类网站的通用解决方案，而 Docusaurus 则专注于将一件事做到极致——成为编写和发布内容的最佳工具。

GraphQL 是 Gatsby 的核心设计，不过构建静态网站时通常不需要 GraphQL 提供的灵活性。

Docusaurus v2+ 的许多设计灵感源自 Gatsby 的优秀特性，它是个很好的替代选择。

[Docz](https://github.com/pedronauck/docz) 是基于 Gatsby 的文档主题，目前功能不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一个流行的混合式 React 框架。虽然能用于构建文档网站，但其设计并非专为文档场景优化，实现 Docusaurus 开箱即用的功能需要更多开发工作。

[Nextra](https://github.com/shuding/nextra) 是基于 Next.js 的静态站点生成器，目前功能不如 Docusaurus 全面。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 与 Docusaurus 有许多相似之处——都专注于内容型网站并提供开箱即用的文档功能。不同之处在于 VitePress 基于 Vue，而 Docusaurus 基于 React。如需 Vue 技术栈的解决方案，VitePress 是不错的选择。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 静态站点生成器，其价值主张与 Docusaurus 相似。

如果您不需要单页应用且不打算使用 React，这是个很好的选择。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主题。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 便于快速创建文档网站，但它不是静态站点生成器且对 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 拥有简洁的设计，曾被许多开源项目采用。随着其产品逐渐商业化，已不再符合开源文档站点的需求（详见 Redux [迁移至 Docusaurus](https://github.com/reduxjs/redux/issues/3161) 的讨论）。

当前 GitBook 仅对开源和非盈利团队免费，而 Docusaurus 对所有人免费。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的静态站点生成器之一——事实上在 Docusaurus 之前，Facebook 多数开源网站都基于 Jekyll 构建！我们致力于提供与 Jekyll 同样简洁的开发体验。

与使用静态生成HTML并通过`<script />`标签添加交互性的方案相比，Docusaurus站点是React应用。借助现代JavaScript生态工具链，我们期望为文档站点设立性能、资源构建流程与优化、以及易用性方面的新标准。
