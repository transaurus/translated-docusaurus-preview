# 概述

本文档将指导您将现有的 Docusaurus 1 站点迁移至 Docusaurus 2。

我们尽可能简化此过程，并提供了迁移命令行工具。

## 主要差异 {#main-differences}

Docusaurus 1 是纯粹的文档站点生成器，使用 React 作为服务端模板引擎，但不在浏览器中加载 React。

Docusaurus 2 经过彻底重构，生成单页应用程序，在浏览器中充分发挥 React 的全部能力。它在保留 Docusaurus 1 核心优势（快速上手、版本化文档和国际化支持）的同时，提供了更高的可定制性。

此外，Docusaurus 2 是一个高性能的静态站点生成器，可快速构建各类内容型网站（如文档、博客、产品落地页和营销页面等）。

虽然我们仍以帮助您完善文档为核心目标，但由于 Docusaurus 2 本质上是 React 应用，现在可用于构建任何类型的网站。Docusaurus 已不仅限于文档网站建设。

## Docusaurus 1 结构 {#docusaurus-1-structure}

您的 Docusaurus 1 站点应具有以下结构：

```bash
├── docs
└── website
    ├── blog
    ├── core
    │   └── Footer.js
    ├── package.json
    ├── pages
    ├── sidebars.json
    ├── siteConfig.js
    └── static
```

## Docusaurus 2 结构 {#docusaurus-2-structure}

迁移完成后，您的 Docusaurus 2 站点可能呈现如下结构：

```sh
├── docs
└── website
    ├── blog
    ├── src
    │   ├── components
    │   ├── css
    │   └── pages
    ├── static
    ├── package.json
    ├── sidebars.json
    ├── docusaurus.config.js
```

:::info

本次迁移不会改变 `/docs` 文件夹位置，但 Docusaurus v2 站点通常将 `/docs` 文件夹置于 `/website` 目录下

迁移至 v2 后，您可自由选择 `/docs` 文件夹的存放位置。

:::

## 迁移流程 {#migration-process}

要获得功能完整的 Docusaurus 2 网站，需迁移以下内容：

- 软件包
- CLI 命令
- 站点配置
- Markdown 文件
- 侧边栏文件
- 页面、组件和 CSS
- 版本化文档
- 国际化支持 🚧

## 自动化迁移流程 {#automated-migration-process}

[迁移命令行工具](./migration-automated.mdx)将为您处理大部分迁移工作。

但部分内容难以自动化处理，您仍需进行手动迁移。

:::note

我们建议先运行迁移命令行工具，再通过手动迁移流程完成剩余部分。

:::

## 手动迁移流程 {#manual-migration-process}

部分迁移内容（特别是页面）无法自动化处理，需要您手动迁移。

[手动迁移指南](./migration-manual.mdx)将提供完整的手动操作步骤。
