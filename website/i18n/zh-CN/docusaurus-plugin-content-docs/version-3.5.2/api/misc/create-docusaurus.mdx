# 📦 create-docusaurus

一个脚手架工具，可帮助您快速搭建功能完备的Docusaurus应用。

## 用法 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name`参数将同时作为站点路径和生成应用的package.json中的`name`字段值。可以是绝对路径或相对于`rootDir`的相对路径。

`template`参数可选值包括：

- `classic`：使用经典模板（推荐）
- `facebook`：使用Facebook/Meta模板（包含Meta特有的配置）
- 以`https://`或`git@`开头的Git仓库URL（可克隆到目标目录）
- 相对于当前工作目录的本地文件路径（其内容将被复制到目标目录）

`rootDir`参数用于解析站点目录的绝对路径，默认为当前工作目录。

:::warning

建议在交互式shell中使用此命令以确保所有功能可用。

:::

## 选项 {#options}

### `-t, --typescript` {#typescript}

当模板参数为预设名称时启用。目前仅`classic`模板提供TypeScript变体。

### `-g, --git-strategy` {#git-strategy}

当模板参数为Git仓库时指定克隆策略，可选值：

- `deep`：保留完整Git历史记录
- `shallow`：使用`--depth=1`浅克隆
- `copy`：浅克隆且不创建Git仓库
- `custom`：输入自定义git clone命令（将提示输入，例如可填写`git clone --depth 10`，工具会自动追加仓库URL和目标目录参数）

### `-p, --package-manager` {#package-manager}

指定包管理器（`npm`/`yarn`/`pnpm`/`bun`）。若未显式指定，Docusaurus将按以下优先级推断：

- 当前工作目录已存在的锁文件（如在现有项目中搭建网站）
- 调用`create-docusaurus`的命令（如`npm init`/`npx`/`yarn create`/`bunx`等）
- 当所有启发式条件均不满足时，进行交互式提示

### `-s, --skip-install` {#skip-install}

若启用此选项，创建应用后不会自动安装依赖。`--package-manager`选项仅在需要安装依赖时生效。
