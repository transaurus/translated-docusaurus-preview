# 文档功能简介

文档功能为用户提供了以层级结构组织Markdown文件的方式。

:::info

完整配置选项请查阅[文档插件API参考文档](./../../api/plugins/plugin-content-docs.mdx)

:::

站点的文档系统分为四个层级（从低到高）：

1. 独立页面
2. 侧边栏
3. 版本
4. 插件实例

本指南将按此顺序介绍：[如何配置独立页面](./docs-create-doc.mdx)，[创建单个或多个侧边栏](./sidebar/index.mdx)，[版本管理](./versioning.mdx)，以及[使用多文档插件实例](./docs-multi-instance.mdx)。

## 纯文档模式 {#docs-only-mode}

新初始化的Docusaurus站点具有以下结构：

```
example.com/                                -> generated from `src/pages/index.js`

example.com/docs/intro                      -> generated from `docs/intro.md`
example.com/docs/tutorial-basics/...        -> generated from `docs/tutorial-basics/...`
...

example.com/blog/2021/08/26/welcome         -> generated from `blog/2021-08-26-welcome/index.md`
example.com/blog/2021/08/01/mdx-blog-post   -> generated from `blog/2021-08-01-mdx-blog-post.mdx`
...
```

所有文档默认通过子路由`docs/`访问。但如果**站点仅包含文档**，或希望将文档置于根目录时该如何处理？

假设您的配置如下：

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          /* docs plugin options */
        },
        blog: {
          /* blog plugin options */
        },
        // ...
      },
    ],
  ],
};
```

要启用纯文档模式，需修改为：

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          routeBasePath: '/', // Serve the docs at the site's root
          /* other docs plugin options */
        },
        // highlight-next-line
        blog: false, // Optional: disable the blog plugin
        // ...
      },
    ],
  ],
};
```

注意：您**无需放弃博客或其他插件功能**，`routeBasePath: '/'`仅改变文档访问路径——从`https://example.com/docs/some-doc`变为根目录下的`https://example.com/some-doc`。若启用博客，仍可通过`blog/`子路由访问。

别忘了通过添加front matter将某个文档设置为根页面（`https://example.com/`）：

```md title="docs/intro.md"
---
# highlight-next-line
slug: /
---

This page will be the home page when users visit https://example.com/.
```

:::warning

若通过`slug: /`将文档设为主页，请删除现有的`./src/pages/index.js`主页文件，否则会出现两个文件映射到同一路由的情况！

:::

调整后的站点结构如下：

```
example.com/                       -> generated from `docs/intro.md`
example.com/tutorial-basics/...    -> generated from `docs/tutorial-basics/...`
...
```

:::tip

Docusaurus同样支持"纯博客模式"。操作方法同上，具体设置请参考[纯博客模式](../../blog.mdx#blog-only-mode)。

:::
