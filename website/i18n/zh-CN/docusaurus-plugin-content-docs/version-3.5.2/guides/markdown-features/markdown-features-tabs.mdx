# 标签页

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-tabs-styles.module.css';

Docusaurus 提供了 `<Tabs>` 组件，通过 [MDX](./markdown-features-react.mdx) 可在 Markdown 中使用：

```jsx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana 🍌
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
    <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
    <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

---

也可通过 `values` 和 `defaultValue` 属性配置标签页：

```jsx
<Tabs
  defaultValue="apple"
  values={[
    {label: 'Apple', value: 'apple'},
    {label: 'Orange', value: 'orange'},
    {label: 'Banana', value: 'banana'},
  ]}>
  <TabItem value="apple">This is an apple 🍎</TabItem>
  <TabItem value="orange">This is an orange 🍊</TabItem>
  <TabItem value="banana">This is a banana 🍌</TabItem>
</Tabs>
```

:::tip

默认情况下所有标签页会在构建时预渲染，搜索引擎可索引隐藏标签页内容。

通过 `<Tabs lazy />` 可仅渲染默认标签页。

:::

## 设置默认标签页 {#displaying-a-default-tab}

默认显示首个标签页。可通过在某个 `<TabItem>` 添加 `default` 属性，或为 `<Tabs>` 设置 `defaultValue="标签值"` 来覆盖此行为。例如上例中，为 `value="apple"` 的标签添加 `default` 属性，或为 `<Tabs>` 设置 `defaultValue="apple"` 均可强制默认展开"Apple"标签页。

若指定的 `defaultValue` 对应值不存在，Docusaurus 会抛出错误。如需默认不展开任何标签页，请使用 `defaultValue={null}`。



## 同步标签页选择状态 {#syncing-tab-choices}

可为同类标签页设置相同的 `groupId` 属性以实现联动选择。例如针对不同操作系统（Windows/macOS）提供操作指南时，用户切换一次即可更新所有相关标签页。注意：
- 选择状态会持久化到 `localStorage`
- 同组所有 `<Tab>` 实例会自动同步状态
- 组ID为全局命名空间

同组标签页的值域无需完全一致。当某组选中其他组不存在的值时，缺失该值的标签组将保持当前状态。例如下例中选择"Linux"时，上方标签组不会变化。

```jsx
// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + C to copy.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + C to copy.</TabItem>
</Tabs>

// highlight-next-line
<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">Use Ctrl + V to paste.</TabItem>
  <TabItem value="mac" label="macOS">Use Command + V to paste.</TabItem>
</Tabs>
```

## 自定义标签页 {#customizing-tabs}

通过 `className` 属性可为 `<Tabs>` 添加自定义CSS类名：

```jsx
// highlight-next-line
<Tabs className="unique-tabs">
  <TabItem value="Apple">This is an apple 🍎</TabItem>
  <TabItem value="Orange">This is an orange 🍊</TabItem>
  <TabItem value="Banana">This is a banana 🍌</TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs className="unique-tabs">
    <TabItem value="Apple">This is an apple 🍎</TabItem>
    <TabItem value="Orange">This is an orange 🍊</TabItem>
    <TabItem value="Banana">This is a banana 🍌</TabItem>
  </Tabs>
</BrowserWindow>

### 自定义标签标题 {#customizing-tab-headings}

通过 `attributes` 字段可为每个标签标题单独配置额外属性。可通过以下两种方式传递属性：
1. 在 `<Tabs>` 的 `values` 属性中声明（类似 `label` 的声明方式）
2. 直接在 `<TabItem>` 的props中声明

```jsx title="some-doc.mdx"
import styles from './styles.module.css';

<Tabs>
  <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
    This is an apple 🍎
  </TabItem>
  <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
    This is an orange 🍊
  </TabItem>
  <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
    This is a banana 🍌
  </TabItem>
</Tabs>
```

```css title="styles.module.css"
.red {
  color: red;
}
.red[aria-selected='true'] {
  border-bottom-color: red;
}

.orange {
  color: orange;
}
.orange[aria-selected='true'] {
  border-bottom-color: orange;
}

.yellow {
  color: yellow;
}
.yellow[aria-selected='true'] {
  border-bottom-color: yellow;
}
```

<BrowserWindow>
  <Tabs>
    <TabItem value="apple" label="Apple" attributes={{className: styles.red}}>
      This is an apple 🍎
    </TabItem>
    <TabItem value="orange" label="Orange" attributes={{className: styles.orange}}>
      This is an orange 🍊
    </TabItem>
    <TabItem value="banana" label="Banana" attributes={{className: styles.yellow}}>
      This is a banana 🍌
    </TabItem>
  </Tabs>
</BrowserWindow>

:::tip

`className` 会与其他默认类名合并使用。你也可以通过自定义 `data-value` 字段（如 `{'data-value': 'apple'}`）配合 CSS 属性选择器实现样式定制：

```css title="styles.module.css"
li[role='tab'][data-value='apple'] {
  color: red;
}
```

:::

## 查询字符串 {#query-string}

支持将选中的标签页状态持久化到 URL 查询参数中。此功能允许您分享已预选特定标签页的页面链接（例如从 Android 应用直接跳转到预选 Android 标签页的文档）。注意：该功能不提供锚点链接，浏览器不会自动滚动到标签位置。

通过 `queryString` 属性启用此功能，并定义要使用的搜索参数名称。

```tsx
// highlight-next-line
<Tabs queryString="current-os">
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString='current-os'>
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

当点击标签页时，URL 末尾会自动添加搜索参数，例如：`?current-os=android` 或 `?current-os=ios`。

:::tip

`queryString` 可与 `groupId` 配合使用。

为方便起见，当 `queryString` 属性设为 `true` 时，将自动使用 `groupId` 的值作为回退参数名。

```tsx
// highlight-next-line
<Tabs groupId="current-os" queryString>
  <TabItem value="android" label="Android">
    Android
  </TabItem>
  <TabItem value="ios" label="iOS">
    iOS
  </TabItem>
</Tabs>
```

<BrowserWindow>
  <Tabs queryString groupId="current-os">
    <TabItem value="android" label="Android">Android</TabItem>
    <TabItem value="ios" label="iOS">iOS</TabItem>
  </Tabs>
</BrowserWindow>

页面加载时，会优先从查询字符串恢复标签选择状态（其次才使用 `groupId` 在 `localStorage` 中的存储值）。

:::
